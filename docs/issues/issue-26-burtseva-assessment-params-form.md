## Описание

Форма ввода параметров объекта недвижимости для оценки: адрес с подсказками, площадь, тип объекта, технические характеристики, валидации, автозаполнение.

## Затронутые роли

- Заявитель

## Экраны / компоненты

- [ ] **Форма параметров объекта** (`/applicant/assessment/new/params`) — поля:
  - Адрес объекта с автодополнением (Nominatim / Яндекс.Адрес)
  - Кадастровый номер (опциональный)
  - Площадь (кв.м) — числовое поле с валидацией диапазона
  - Тип объекта — выпадающий список (квартира / дом / участок / коммерческая / другое)
  - Характеристики по типу:
    - Квартира / дом: этаж, этажность здания, состояние (отличное/хорошее/удовлетворительное/плохое), год постройки, наличие балкона/лоджии
    - Участок: категория земли, ВРИ, коммуникации
- [ ] **Автозаполнение по кадастровому номеру** — при вводе номера подтянуть доступные данные из внешнего API
- [ ] **Inline-валидации** — сообщения об ошибках под каждым полем в реальном времени

## Технические требования

- Маршрут: встроен в многошаговый флоу оценки (`/applicant/assessment/new`, шаг 2)
- Реактивная форма Angular с динамическими полями по типу объекта (`switchMap` / `formGroup` пересборка)
- Адресное автодополнение: debounce 400 мс, минимум 3 символа
- RPC: `AssessmentService.Create` / `Update` (создать черновик, обновлять по шагам)

## Acceptance criteria

- [ ] При вводе адреса появляются подсказки от геокодера
- [ ] Поля характеристик меняются динамически при смене типа объекта
- [ ] Все обязательные поля валидируются с понятными сообщениями
- [ ] Черновик сохраняется при переходе между шагами
- [ ] Автозаполнение по кадастровому номеру подставляет известные поля
