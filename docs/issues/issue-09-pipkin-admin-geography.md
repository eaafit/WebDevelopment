## Описание

Интерактивная карта объектов оценки в панели администратора: маркеры по адресам заявок, фильтрация, переход к карточке заявки, геокодирование.

## Затронутые роли

- Администратор

## Экраны / компоненты

- [ ] **Страница карты** (`/admin/geography`) — карта Leaflet с маркерами по всем объектам оценки
- [ ] **Маркеры** — отображают адрес объекта и статус заявки; цветовая кодировка по статусу
- [ ] **Панель фильтров** — фильтр по статусу заявки, периоду дат
- [ ] **Попап маркера** — краткая сводка: адрес, статус, дата заявки; кнопка «Открыть заявку» → переход к `/admin/orders/:id`
- [ ] **Геокодирование адреса** — если у заявки нет координат, выполнить геокодирование через внешний API (Nominatim / Яндекс)

## Технические требования

- Маршрут: `/admin/geography`
- Защита: `roleGuard(UserRole.Admin)` из `@notary-portal/ui`
- Leaflet: подключить через `npm install leaflet @types/leaflet` или через CDN в `index.html`
- Использовать `AssessmentService.List` для получения заявок с адресами
- Геокодирование: REST-запрос к Nominatim (`https://nominatim.openstreetmap.org/search`) или Яндекс.Геокодер
- Кластеризация маркеров при большом количестве точек (`leaflet.markercluster`)

## Acceptance criteria

- [ ] Карта загружается и отображает маркеры по всем заявкам с адресами
- [ ] Фильтры по статусу и периоду обновляют маркеры без перезагрузки страницы
- [ ] Клик на маркер открывает попап с кратким описанием
- [ ] Кнопка в попапе ведёт на карточку заявки
- [ ] Если адрес без координат — выполняется геокодирование и маркер добавляется на карту
