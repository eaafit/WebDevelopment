## Описание

UI-модуль для массовой загрузки фото и документов объекта недвижимости при подаче на оценку: сортировка, предпросмотр, метки, требования к качеству.

## Затронутые роли

- Заявитель
- Нотариус

## Экраны / компоненты

- [ ] **Зона массовой загрузки** — множественный выбор файлов, drag&drop с поддержкой нескольких файлов одновременно
- [ ] **Галерея загруженных файлов** — сетка миниатюр (фото) / иконок (PDF), возможность перетаскивания для изменения порядка (сортировка)
- [ ] **Метки фотографий** — для каждого файла выбрать метку: «Фасад», «План», «Правоустанавливающий документ», «Другое»
- [ ] **Предпросмотр** — клик открывает lightbox (для фото) или PDF-viewer
- [ ] **Индикатор требований к качеству** — предупреждения: минимальное разрешение, ориентация (горизонтальная/вертикальная), читаемость (для PDF — размер файла)
- [ ] **Кнопка удаления** с подтверждением для каждого файла

## Технические требования

- Расположить компонент в `libs/web/applicant/src/lib/features/assessment/upload/` и `/notary/`
- Базируется на общем `FileUploadComponent` (issue-14, Головатый Константин) — расширить под нужды модуля оценки
- Сортировка drag&drop: `@angular/cdk/drag-drop` (`CdkDragDrop`)
- Проверка качества изображений: через `Image` объект браузера (размеры пикселей)
- RPC: `DocumentService.Create` с `documentType` из enum `DocumentType`

## Acceptance criteria

- [ ] Можно загрузить несколько файлов одновременно (фото + PDF)
- [ ] Каждому файлу присваивается метка (обязательно для отправки)
- [ ] Порядок файлов изменяется перетаскиванием
- [ ] Фото с разрешением ниже минимального получают предупреждение
- [ ] Предпросмотр открывается по клику на миниатюру
- [ ] Нельзя отправить форму без меток на всех файлах
