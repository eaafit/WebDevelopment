## Описание

Многошаговая форма подачи заявки на оценку наследственного имущества: ввод данных об объекте, выбор типа имущества, прикрепление документов, согласие на обработку, отправка.

## Затронутые роли

- Заявитель (также Гость с последующей регистрацией)

## Экраны / компоненты

- [ ] **Шаг 1 — Данные о наследстве** — поля: ФИО наследодателя, дата смерти, нотариус (опц.), номер наследственного дела
- [ ] **Шаг 2 — Параметры объекта** — тип имущества (недвижимость / ТС / иное), адрес / идентификатор, кадастровый номер (опц.)
- [ ] **Шаг 3 — Прикрепление документов** — загрузка: свидетельство о смерти, правоустанавливающие документы, фото объекта; drag&drop
- [ ] **Шаг 4 — Согласие и отправка** — чекбоксы согласий, кнопка «Отправить заявку»
- [ ] **Страница успешной подачи** — подтверждение с номером заявки и кнопкой «Мои заявки»

## Технические требования

- Маршрут: `/applicant/orders/new`
- Защита: `roleGuard(UserRole.Applicant)` (или публичный с редиректом на регистрацию для гостей)
- Реактивная форма Angular со сквозной валидацией
- Черновик формы: сохранять в `sessionStorage` для восстановления при перезагрузке
- RPC: `AssessmentService.Create`, `DocumentService.Create`
- Использовать общий компонент загрузки файлов (issue-15, Головатый Константин)

## Acceptance criteria

- [ ] Форма состоит из 4 шагов с индикатором прогресса
- [ ] Нельзя перейти к следующему шагу без заполнения обязательных полей
- [ ] Файлы валидируются по формату (PDF, JPG, PNG) и размеру (макс. 20 МБ)
- [ ] При незапланированном уходе данные формы восстанавливаются
- [ ] После успешной отправки пользователь видит номер заявки
