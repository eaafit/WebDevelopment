## Описание

Лента заказов с отслеживанием статусов: хронологический таймлайн переходов, возможность повторить заказ, фильтры, уведомления об изменениях.

## Затронутые роли

- Заявитель
- Нотариус

## Экраны / компоненты

- [ ] **Лента заказов** (`/applicant/assessment/history`, `/notary/assessment/history`) — карточки заказов в хронологическом порядке: адрес, тип объекта, дата, текущий статус
- [ ] **Таймлайн статусов** внутри карточки/страницы заказа — вертикальная шкала: Создан → В очереди → В работе → Завершён (или Отменён), с датами каждого перехода
- [ ] **Кнопка «Повторить заказ»** — создать новый заказ с теми же параметрами (предзаполнить форму)
- [ ] **Фильтры** — по статусу, диапазону дат, типу объекта
- [ ] **Уведомления об изменениях** — toast/badge при обновлении статуса заказа (polling или WebSocket)

## Технические требования

- Маршруты: `/applicant/assessment/history`, `/notary/assessment/history`
- Защита по роли
- RPC: `AssessmentService.List` (с пагинацией и фильтрами), `AssessmentService.Get`
- Таймлайн: компонент `TimelineComponent` — переиспользуемый, `@Input() steps: TimelineStep[]`
- «Повторить заказ»: скопировать параметры через router state и открыть форму из issue-26

## Acceptance criteria

- [ ] Лента заказов отображается с актуальными статусами
- [ ] Таймлайн внутри заказа показывает даты всех переходов статусов
- [ ] «Повторить заказ» открывает форму с предзаполненными параметрами прошлого заказа
- [ ] Фильтры корректно сужают выборку
- [ ] Новый статус отображается без полной перезагрузки страницы
