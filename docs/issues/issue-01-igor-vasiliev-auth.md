## Описание

Реализация экранов аутентификации: создание аккаунта, вход в систему, сброс пароля по email или телефону.

## Затронутые роли

- Гость
- Заявитель
- Нотариус
- Администратор

## Экраны / компоненты

- [ ] **Страница регистрации** — форма с полями: email, пароль, подтверждение пароля, ФИО, телефон, выбор роли (Заявитель / Нотариус)
- [ ] **Страница входа** — форма email + пароль, ссылка «Забыл пароль»
- [ ] **Страница восстановления пароля** — запрос сброса по email
- [ ] **Страница сброса пароля** — форма нового пароля по токену из письма

## Технические требования

- Использовать `AuthService` из `@notary-portal/guest` (методы `login`, `register`)
- После успешного входа — навигация по роли через `USER_ROLE_HOME` из `TokenStore`
- Реактивные формы Angular (`ReactiveFormsModule`), валидация на стороне клиента
- Отображать `authService.error()` и `authService.loading()` (сигналы)
- Маршруты: `/auth`, `/auth/register`, `/auth/forgot-password`, `/auth/reset-password`

## Acceptance criteria

- [ ] Пользователь может зарегистрироваться как Заявитель или Нотариус
- [ ] После регистрации/входа происходит автоматический редирект в нужный кабинет
- [ ] При неверных данных отображается понятное сообщение об ошибке
- [ ] Форма заблокирована на время запроса (spinner / disabled)
- [ ] Восстановление пароля отправляет письмо и показывает подтверждение
