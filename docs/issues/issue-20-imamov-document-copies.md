## Описание

Экраны для запроса, оплаты и выдачи копий нотариальных документов: форма запроса, прикрепление оснований, расчёт стоимости, оплата, статусы обработки, получение готовых копий.

## Затронутые роли

- Заявитель
- Нотариус (в зависимости от бизнес-модели)

## Экраны / компоненты

- [ ] **Страница «Копии документов»** (`/applicant/copies`, `/notary/copies`) — список запросов на копии: статус, дата, тип документа
- [ ] **Форма запроса копии** (`/applicant/copies/new`) — выбор типа документа, номер дела/заявки, прикрепление оснований (доверенность, свидетельство и т.д.), комментарий
- [ ] **Расчёт стоимости** — после заполнения формы автоматически отображается расчёт стоимости копии
- [ ] **Шаг оплаты** — переход к оплате (интеграция с компонентом оплаты из issue-18)
- [ ] **Статус «В обработке»** — страница/badge с таймером ожидания
- [ ] **Страница «Готово»** — ссылки на скачивание готовых копий, статус «Выдано»

## Технические требования

- Маршруты: `/applicant/copies`, `/applicant/copies/new`, `/applicant/copies/:id`
- Защита: `roleGuard(UserRole.Applicant)` / `roleGuard(UserRole.Notary)`
- RPC: `DocumentService.Create`, `DocumentService.Get`, `DocumentService.List`
- Загрузка оснований: использовать общий компонент (issue-14, Головатый Константин)
- Расчёт стоимости: запрос к бэкенду (согласовать endpoint)

## Acceptance criteria

- [ ] Пользователь может создать запрос на копию с прикреплёнными основаниями
- [ ] Система рассчитывает и отображает стоимость перед оплатой
- [ ] После оплаты статус запроса меняется на «В обработке»
- [ ] Когда копия готова — статус меняется на «Готово», появляется ссылка на скачивание
- [ ] История запросов доступна на странице `/applicant/copies`
