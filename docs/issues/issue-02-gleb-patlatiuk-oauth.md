## Описание

Реализация OAuth-авторизации через внешние провайдеры: VK, Google, Apple, Yandex. Форма подтверждения контакта (email / телефон) после OAuth-входа.

## Затронутые роли

- Гость
- Заявитель
- Нотариус
- Администратор

## Экраны / компоненты

- [ ] **Кнопки OAuth на странице входа/регистрации** — иконки и подписи VK / Google / Apple / Yandex
- [ ] **Страница подтверждения контакта** — ввод кода из SMS или ссылки из email после OAuth-входа
- [ ] **Модальное/страничное уведомление** об успешной привязке аккаунта

## Технические требования

- Интегрировать OAuth-редиректы (согласовать callback-URL с бэкендом)
- После OAuth-callback получить токены через `AuthService` (метод согласовать с бэкендом)
- Хранить токены через `TokenStore` из `@notary-portal/ui`
- Форма подтверждения контакта: ввод 4–6 значного кода, таймер повторной отправки

## Acceptance criteria

- [ ] Клик по кнопке VK/Google/Apple/Yandex открывает OAuth-окно провайдера
- [ ] После успешной OAuth-авторизации пользователь попадает в соответствующий кабинет
- [ ] Если аккаунт новый — предлагается форма подтверждения контакта перед входом
- [ ] Код подтверждения истекает через 5 минут, доступна повторная отправка
