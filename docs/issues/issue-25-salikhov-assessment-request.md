## Описание

UI для отправки объекта на оценку: выбор модели оценки, уровня срочности, подтверждение параметров, отслеживание этапов обработки.

## Затронутые роли

- Заявитель
- Нотариус

## Экраны / компоненты

- [ ] **Страница «Отправить на оценку»** — форма с параметрами:
  - Выбор модели оценки (если несколько доступны)
  - Уровень срочности (стандарт / срочно) — влияет на стоимость и сроки
  - Дополнительные комментарии
- [ ] **Сводка параметров** — экран подтверждения: объект, загруженные документы, выбранная модель, стоимость, срок; кнопки «Подтвердить» / «Назад»
- [ ] **Шкала этапов** — после отправки: визуализация прогресса оценки (Получена → В очереди → Обрабатывается → Завершена), с датами переходов
- [ ] **Уведомление о смене этапа** — inline-статус или toast при обновлении

## Технические требования

- Маршрут: `/applicant/assessment/request/:assessmentId`, `/notary/assessment/request/:assessmentId`
- Защита по роли
- RPC: `AssessmentService.Update` (передать параметры модели/срочности), `AssessmentService.Get` (polling статуса)
- Polling статуса: каждые 10–30 с пока оценка не завершена
- Шкала этапов: компонент-степпер (`StepperComponent` или кастомный)

## Acceptance criteria

- [ ] Пользователь может выбрать модель и срочность
- [ ] На экране подтверждения отображаются итоговая стоимость и срок
- [ ] После отправки виден прогресс этапов
- [ ] При смене статуса шкала обновляется без перезагрузки страницы
