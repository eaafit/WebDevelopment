## Описание

Реализация личного кабинета нотариуса: просмотр и управление заказами, фильтрация, смена статусов, управление подпиской, работа с документами.

## Затронутые роли

- Нотариус

## Экраны / компоненты

- [ ] **Главная кабинета** (`/notary`) — дашборд: активные заказы, уведомления, ссылки на быстрые действия
- [ ] **Список заказов** (`/notary/orders`) — таблица заказов: статус, дата, заявитель, объект; поиск, фильтры
- [ ] **Карточка заказа** (`/notary/orders/:id`) — детали заявки, кнопка «Взять в работу», смена статуса, документы заявителя
- [ ] **Управление статусами** — выпадающий список / кнопки перехода между статусами (получен → в работе → завершён / отменён)
- [ ] **Страница подписки** (`/notary/subscription`) — текущий тариф, срок, кнопка оплаты/продления

## Технические требования

- Маршруты: `/notary`, `/notary/orders`, `/notary/orders/:id`, `/notary/subscription`
- Защита: `roleGuard(UserRole.Notary)` из `@notary-portal/ui`
- RPC: `AssessmentService` (методы `List`, `Get`, `Update`), `DocumentService.List`
- Пагинация, debounced-поиск

## Acceptance criteria

- [ ] Нотариус видит только заказы, доступные его роли
- [ ] «Взять в работу» меняет статус заявки и блокирует кнопку
- [ ] Смена статуса доступна только для разрешённых переходов
- [ ] Страница подписки показывает актуальный тариф
- [ ] Незалогиненный / не та роль → редирект

## Связанные файлы

- Proto-контракты: `AssessmentService`, `DocumentService`, `PaymentService`
