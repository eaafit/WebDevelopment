## Описание

Экраны оплаты для нотариусов и заявителей: выбор тарифного плана, ввод реквизитов, применение промокода, подтверждение оплаты, просмотр чеков/счетов, история платежей.

## Затронутые роли

- Нотариус
- Заявитель (если платит за доступ)

## Экраны / компоненты

- [ ] **Страница выбора тарифа** (`/notary/subscription`, `/applicant/payments/plans`) — карточки тарифных планов: название, цена, период, включённые функции; кнопка «Выбрать»
- [ ] **Форма оплаты** — поля: номер карты, срок, CVV (или интеграция с платёжным виджетом); поле ввода промокода с кнопкой «Применить»
- [ ] **Страница подтверждения оплаты** — сводка: тариф, сумма со скидкой (если есть промокод), кнопки «Оплатить» / «Назад»
- [ ] **Квитанция / чек** — страница после успешной оплаты с деталями транзакции и кнопкой «Скачать PDF»
- [ ] **История платежей** (`/notary/transactions`, `/applicant/payments`) — таблица транзакций (переиспользовать `TransactionTableComponent` из `@notary-portal/ui`)

## Технические требования

- Маршруты: `/notary/subscription`, `/notary/transactions`, `/applicant/payments`, `/applicant/payments/plans`
- Защита по роли: `roleGuard(UserRole.Notary)` / `roleGuard(UserRole.Applicant)`
- RPC: `PaymentService.CreatePayment`, `PaymentService.GetPaymentHistory`, `PaymentService.GetSubscription`
- Промокод: отправить на бэкенд и получить пересчитанную сумму
- Платёжная форма: ввод карточных данных только через безопасный виджет (Stripe / ЮKassa — согласовать с командой)

## Acceptance criteria

- [ ] Пользователь видит список доступных тарифов с ценами
- [ ] Промокод применяется и пересчитывает итоговую сумму
- [ ] После успешной оплаты отображается чек с деталями транзакции
- [ ] Чек можно скачать в PDF
- [ ] История платежей показывает все транзакции с пагинацией и фильтрами
