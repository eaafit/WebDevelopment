## Описание

Экраны чата поддержки: создание тикетов, переписка с поддержкой, прикрепление вложений, отображение SLA-статусов.

> **Разработчики:** Васорин Иван и Говор Сергей — работают над чатом совместно.  
> Васорин Иван: базовый чат/тикеты, вложения, SLA.  
> Говор Сергей (issue-22): база знаний/FAQ и поиск по статьям.

## Затронутые роли

- Все роли (поддержку ведёт внутренняя роль / администратор)

## Экраны / компоненты

- [ ] **Список тикетов** (`/support`) — список обращений пользователя: тема, статус (открыт/в работе/закрыт), дата, SLA-индикатор
- [ ] **Страница тикета** (`/support/:id`) — переписка в формате чата: сообщения, дата/время, роль отправителя
- [ ] **Форма нового тикета** — тема, описание, приоритет, прикрепление файлов (до 5 шт.)
- [ ] **Вложения** — отображение прикреплённых файлов: предпросмотр изображений, ссылки на скачивание PDF
- [ ] **SLA-индикаторы** — таймер оставшегося времени по SLA, цветовая кодировка (зелёный/жёлтый/красный)

## Технические требования

- Маршруты: `/support`, `/support/new`, `/support/:id`
- Доступно всем ролям (guard: только аутентификация)
- Polling или WebSocket для обновления новых сообщений (согласовать с бэкендом)
- Загрузка файлов: использовать общий компонент (issue-15)
- RPC: согласовать endpoint для тикетов с бэкендом (proto не определён — создать задачу)

## Acceptance criteria

- [ ] Пользователь видит список своих тикетов с актуальными статусами
- [ ] Можно создать новый тикет с описанием и вложениями
- [ ] В открытом тикете видна переписка в хронологическом порядке
- [ ] SLA-таймер отображается и обновляется в реальном времени
- [ ] Уведомление о новом ответе поддержки приходит без перезагрузки
