## Описание

Переиспользуемый компонент **`lib-support-ai-chat-widget`** в shared UI: компактная панель с полем ввода вопроса и кнопкой отправки, плавающий режим (FAB) и inline-режим. Событие **`messageSubmit`** передаёт текст вопроса родителю; интеграция с LLM/RPC выполняется на следующих этапах (серверный вызов, отображение ответа в теле панели).

Связь с существующей постановкой: **issue-11** (полноценный чат/тикеты), **issue-21** (FAQ в поддержке). Данная задача закрывает минимальный UI-слой и контракт «вопрос → наружу».

## Затронутые роли

- Гость (публичная зона) — виджет подключён в `guest` layout
- Далее: заявитель, нотариус, админ — при необходимости вставить тот же компонент

## Экраны / компоненты

- [x] **Виджет `SupportAiChatWidget`** (`libs/web/shared/ui/src/lib/support-ai-chat/`)
  - Режим `floating`: кнопка внизу справа, раскрывающаяся панель
  - Режим `inline`: только панель без FAB
  - Заголовок, подзаголовок, плейсхолдер, кнопка отправки — через `@Input`
  - Enter / Shift+Enter в поле ввода (см. компонент)
- [x] **Интеграция в гостевую зону** — `(messageSubmit)` → заглушка до подключения API
- [ ] **Вызов бэкенда / ИИ** — RPC или HTTP из обработчика `messageSubmit`
- [ ] **Лента сообщений** вместо статичной подсказки в теле панели
- [ ] **Состояния загрузки / ошибки** при запросе к агенту

## Технические требования

- Экспорт из `@notary-portal/ui`
- Стили: CSS-переменные портала (`--primary`, `--bg-card`, …) с fallback в `:host`
- Секреты и вызовы LLM только на API, не из браузера
