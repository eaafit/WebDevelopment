## Описание

Реализация главного раздела панели администратора: просмотр и управление заказами, фильтрация, смена статусов, управление подпиской пользователей.

## Затронутые роли

- Администратор

## Экраны / компоненты

- [ ] **Главная панели** (`/admin`) — сводный дашборд: ключевые метрики, последние заказы, быстрые ссылки
- [ ] **Список заказов** (`/admin/orders`) — таблица всех заявок системы; поиск, фильтры (статус, дата, нотариус, заявитель)
- [ ] **Карточка заказа** (`/admin/orders/:id`) — полный просмотр деталей, ручная смена статуса, документы, история изменений
- [ ] **Управление статусами** — кнопки / выпадающий список переходов между всеми статусами
- [ ] **Управление подпиской** — просмотр текущей подписки пользователя, возможность назначить / продлить

## Технические требования

- Маршруты: `/admin`, `/admin/orders`, `/admin/orders/:id`
- Защита: `roleGuard(UserRole.Admin)` из `@notary-portal/ui`
- RPC: `AssessmentService` (все методы), `UserService.ListUsers`, `PaymentService`
- Серверная пагинация, debounced-поиск

## Acceptance criteria

- [ ] Администратор видит все заказы в системе
- [ ] Доступны фильтры по статусу, дате, нотариусу, заявителю
- [ ] Можно вручную изменить статус любого заказа
- [ ] Доступно управление подпиской конкретного пользователя
- [ ] Без роли Admin — редирект на `/`

## Связанные файлы

- `libs/web/admin/src/lib/`
- Proto-контракты: `AssessmentService`, `UserService`, `PaymentService`
