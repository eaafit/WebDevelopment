## Описание

Формы управления платежами в панели администратора: создание, редактирование, детальный просмотр и модальное окно подтверждения удаления.

> **Разработчики:** Трушин Евгений (issue-16) — список. Липовцев Родион — формы и удаление.

## Затронутые роли

- Администратор

## Экраны / компоненты

- [ ] **Страница деталей платежа** (`/admin/payments/:id`) — все поля: ID, пользователь, тип, статус, сумма, дата, метод оплаты, связанный заказ/подписка, вложения
- [ ] **Форма создания платежа** (`/admin/payments/new`) — поля: пользователь (поиск), тип, сумма, валюта, описание, дата; кнопки «Сохранить» / «Отмена»
- [ ] **Форма редактирования платежа** (`/admin/payments/:id/edit`) — предзаполненные поля из деталей, возможность изменить статус и описание
- [ ] **Модальное окно удаления** — подтверждение с именем платежа, предупреждение о необратимости, кнопки «Удалить» / «Отмена»

## Технические требования

- Маршруты: `/admin/payments/:id`, `/admin/payments/new`, `/admin/payments/:id/edit`
- Защита: `roleGuard(UserRole.Admin)` из `@notary-portal/ui`
- Реактивные формы Angular с валидацией
- RPC: `PaymentService` (методы согласовать с бэкендом — Create/Update/Delete могут отсутствовать в текущем proto)
- Модальное окно: отдельный Angular-компонент `ConfirmDeleteModalComponent`

## Acceptance criteria

- [ ] Страница деталей показывает все поля платежа
- [ ] Форма создания корректно валидирует обязательные поля перед отправкой
- [ ] Форма редактирования предзаполнена актуальными данными
- [ ] Модальное окно удаления требует явного подтверждения
- [ ] После успешного удаления происходит редирект на список платежей
