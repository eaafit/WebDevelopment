## Описание

Таблица-список всех заполненных запросов на оценку недвижимости: просмотр, редактирование, удаление записей. Работает для ролей Заявитель и Нотариус.

## Затронутые роли

- Заявитель
- Нотариус

## Экраны / компоненты

- [ ] **Таблица запросов** (`/applicant/assessment/history`, `/notary/assessment/history`) — колонки: адрес, площадь, тип объекта, статус, дата создания, действия (просмотр / редактировать / удалить)
- [ ] **Строка таблицы** — краткая сводка параметров: адрес (подсказка), площадь, тип, этажность, состояние
- [ ] **Фильтры и поиск** — поиск по адресу, фильтр по типу объекта, статусу, диапазону дат
- [ ] **Действия в строке**:
  - «Просмотр» → переход к деталям
  - «Редактировать» → открыть форму параметров (issue-26) для изменения
  - «Удалить» → подтверждение → удаление черновика (только если статус Draft)
- [ ] **Пагинация** — серверная, с выбором числа строк на страницу (10/25/50)

## Технические требования

- Маршруты: `/applicant/assessment/history`, `/notary/assessment/history`
- Защита по роли
- RPC: `AssessmentService.List` с пагинацией и фильтрами, `AssessmentService.Cancel` (для удаления черновика)
- Реактивное обновление: при изменении фильтров делать новый запрос

## Acceptance criteria

- [ ] Таблица показывает все запросы с корректными данными
- [ ] Поиск и фильтры работают с debounce 300 мс
- [ ] Клик «Редактировать» открывает форму с предзаполненными данными
- [ ] «Удалить» доступно только для черновиков (Draft)
- [ ] Пагинация корректно работает при любом количестве записей
