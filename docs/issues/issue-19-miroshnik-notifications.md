## Описание

Экраны системы уведомлений: лента входящих уведомлений, фильтры прочитанных/непрочитанных, история событий пользователя, настройки каналов доставки (email / push).

## Затронутые роли

- Заявитель
- Нотариус
- Администратор

## Экраны / компоненты

- [ ] **Колокольчик / badge** в шапке кабинета — количество непрочитанных, клик открывает дропдаун с последними 5 уведомлениями
- [ ] **Страница уведомлений** (`/<role>/notifications`) — полная лента уведомлений; фильтры: «все», «непрочитанные», по типу
- [ ] **Карточка уведомления** — иконка типа, заголовок, краткий текст, дата, кнопка «Отметить прочитанным»
- [ ] **Кнопка «Прочитать все»** — bulk-mark-as-read
- [ ] **История событий** — раздел с хронологической лентой ключевых событий по аккаунту (заявки созданы/изменены, платежи и т.д.)
- [ ] **Настройки каналов** (`/<role>/notifications/settings`) — тогглы: email-уведомления вкл/выкл по типу события, push-уведомления (если поддерживается браузером)

## Технические требования

- Маршруты: `/<role>/notifications`, `/<role>/notifications/settings`
- Защита по роли соответствующего кабинета
- RPC: `NotificationService` (методы: `ListNotifications`, `MarkAsRead`, `MarkAllAsRead`, `DeleteNotification`)
- Badge обновляется по polling (интервал 30 с) или WebSocket
- Настройки сохраняются на бэкенд (согласовать endpoint)

## Acceptance criteria

- [ ] Badge в шапке показывает актуальное количество непрочитанных
- [ ] Дропдаун показывает последние 5 уведомлений с кнопкой «Все уведомления»
- [ ] На странице уведомлений работает фильтр «непрочитанные»
- [ ] «Прочитать все» отмечает все уведомления одним действием
- [ ] Настройки каналов сохраняются и применяются к новым уведомлениям
