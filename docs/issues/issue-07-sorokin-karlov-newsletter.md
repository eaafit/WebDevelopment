## Описание

Раздел панели администратора для просмотра списка рассылки и отправки email-рассылок пользователям системы.

> **Разработчики:** Сорокин Д.Э. и Карлов И.А. (DaEtoJostka) — реализуют совместно.  
> Карлов И.А. отвечает за функциональную часть данного issue.

## Затронутые роли

- Администратор

## Экраны / компоненты

- [ ] **Список рассылки** (`/admin/newsletter`) — таблица: email, ФИО, дата подписки, статус подписки (активна/отписан); фильтр, поиск, пагинация
- [ ] **Форма создания рассылки** — выбор аудитории (все / по роли / выбранные), тема письма, HTML/текстовый редактор тела письма, кнопка «Предпросмотр» и «Отправить»
- [ ] **Страница истории рассылок** — список отправленных кампаний: дата, тема, кол-во получателей, статус доставки

## Технические требования

- Маршруты: `/admin/newsletter`, `/admin/newsletter/new`
- Защита: `roleGuard(UserRole.Admin)` из `@notary-portal/ui`
- Редактор письма: встроенный `<textarea>` или WYSIWYG (согласовать)
- RPC / REST: согласовать с бэкендом endpoint для отправки рассылки
- Подтверждение перед массовой отправкой (модальное окно)

## Acceptance criteria

- [ ] Администратор видит полный список подписчиков с возможностью поиска
- [ ] Можно выбрать аудиторию и составить письмо
- [ ] Перед отправкой доступен предпросмотр
- [ ] После отправки отображается подтверждение и запись в истории
- [ ] История показывает статус доставки каждой кампании
