## Описание

Реализация поиска по статьям справочника с расширенными фильтрами: автор, логика поиска (AND/OR, LEFT/RIGHT, CONTAINS/EQUAL).

## Затронутые роли

- Нотариус
- Заявитель (если оплатил доступ)

## Экраны / компоненты

- [ ] **Строка поиска** — текстовое поле с debounce (300 мс), интеграция на главной `/faq` и на страницах категорий
- [ ] **Панель расширенных фильтров** — раскрывающаяся панель с опциями:
  - Логика поиска: `OR`, `AND`, `LEFT`, `RIGHT`, `CONTAINS`, `EQUAL`
  - Фильтр по автору статьи (выпадающий список авторов)
- [ ] **Страница результатов поиска** (`/faq/search?q=...`) — список найденных статей с подсветкой совпадений в заголовке/аннотации
- [ ] **Пустое состояние** — экран «Ничего не найдено» с рекомендацией популярных статей

## Технические требования

- Маршрут: `/faq/search` с query-параметрами `q`, `logic`, `author`
- Debounce на поиск: 300 мс
- Подсветка совпадений: pipe или компонент-хелпер (highlight)
- RPC: согласовать endpoint поиска с бэкендом (передавать логику как параметр)
- URL должен быть share-able: сохранять параметры поиска в query string

## Acceptance criteria

- [ ] Поиск запускается автоматически через 300 мс после прекращения ввода
- [ ] Фильтр по логике (`AND`/`OR`/`CONTAINS` и т.д.) влияет на результаты
- [ ] Фильтр по автору ограничивает выборку статьями конкретного автора
- [ ] Результаты отображают подсветку совпавших слов
- [ ] При отсутствии результатов отображается экран «Ничего не найдено»
- [ ] URL с параметрами поиска можно скопировать и открыть — результаты воспроизведутся
