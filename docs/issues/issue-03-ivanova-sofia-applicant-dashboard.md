## Описание

Реализация личного кабинета заявителя: подача заявки, отслеживание статуса, управление документами.

## Затронутые роли

- Заявитель

## Экраны / компоненты

- [ ] **Главная кабинета** — дашборд с краткой сводкой: активные заявки, последние уведомления, быстрые действия
- [ ] **Список заявок** (`/applicant/orders`) — таблица со статусами, поиск, фильтры по статусу/дате
- [ ] **Карточка заявки** (`/applicant/orders/:id`) — детали заявки, текущий статус, прикреплённые документы, комментарии
- [ ] **Загрузка / замена документов** — внутри карточки заявки, drag&drop или кнопка выбора файла
- [ ] **Комментарии / заметки** — форма добавления заметки к заявке, лента предыдущих комментариев

## Технические требования

- Маршруты: `/applicant`, `/applicant/orders`, `/applicant/orders/:id`
- Все маршруты защищены `roleGuard(UserRole.Applicant)` из `@notary-portal/ui`
- RPC-вызовы через `AssessmentService` и `DocumentService` (из `@notary-portal/api-contracts`)
- Клиент создаётся через `inject(RPC_TRANSPORT)` из `@notary-portal/ui`
- Пагинация списка заявок

## Acceptance criteria

- [ ] Заявитель видит все свои заявки с актуальными статусами
- [ ] Можно открыть карточку заявки и просмотреть детали
- [ ] Можно прикрепить новый документ или заменить существующий
- [ ] Комментарии сохраняются и отображаются в хронологическом порядке
- [ ] Незалогиненный пользователь перенаправляется на `/auth`
