## Описание

Полное управление пользователями и заказами в панели администратора: создание, редактирование, блокировка аккаунтов, управление ролями и правами, ручные корректировки заказов, модерация загруженных файлов.

## Затронутые роли

- Администратор

## Экраны / компоненты

- [ ] **Список пользователей** (`/admin/users`) — таблица: ФИО, email, роль, статус (активен/заблокирован), дата регистрации; поиск, фильтр по роли, пагинация
- [ ] **Карточка пользователя** (`/admin/users/:id`) — профиль, история заказов, текущая подписка, кнопки «Заблокировать», «Изменить роль», «Сбросить пароль»
- [ ] **Форма создания / редактирования пользователя** — ФИО, email, телефон, роль, статус
- [ ] **Список заказов** (`/admin/orders`) — все заказы системы; фильтры, поиск, смена статуса
- [ ] **Ручные корректировки заказа** — изменение полей/статуса, добавление комментария администратора
- [ ] **Модерация файлов** (`/admin/files`) — очередь файлов на проверке: предпросмотр, кнопки «Принять» / «Отклонить», причина отклонения

## Технические требования

- Маршруты: `/admin/users`, `/admin/users/:id`, `/admin/users/new`, `/admin/files`
- Защита: `roleGuard(UserRole.Admin)` из `@notary-portal/ui`
- RPC: `UserService` (все методы: `GetUserById`, `UpdateProfile`, `ListUsers`), `AssessmentService`, `DocumentService`
- Модальные окна подтверждения для блокировки / удаления
- Загрузка файлов: использовать общий компонент из задачи Головатого Константина (issue-15)

## Acceptance criteria

- [ ] Администратор видит полный список пользователей с фильтрами
- [ ] Можно заблокировать/разблокировать пользователя с подтверждением
- [ ] Роль пользователя можно изменить через форму
- [ ] Файлы на модерации отображаются с предпросмотром PDF/изображений
- [ ] Можно принять или отклонить файл с указанием причины
- [ ] Действия логируются (отображаются в истории заказа/пользователя)

## Связанные файлы

- `libs/web/admin/src/lib/features/files/`
