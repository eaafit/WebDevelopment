## Описание

Раздел платежей в панели администратора: список платежей/транзакций, формы создания/редактирования платежа, просмотр деталей, удаление.

> **Разработчики:** Трушин Евгений и Липовцев Родион — работают над разными частями раздела.  
> **Трушин Евгений:** список платежей и транзакций.  
> **Липовцев Родион (issue-17):** формы создания/редактирования и модальное окно удаления.

## Затронутые роли

- Администратор

## Экраны / компоненты

- [ ] **Список платежей** (`/admin/payments`) — таблица: ID транзакции, пользователь, сумма, валюта, тип, статус, дата; поиск, фильтры по статусу/дате/типу, пагинация
- [ ] **Фильтры** — по статусу (Pending / Completed / Failed / Refunded), типу (Subscription / Assessment / DocumentCopy), диапазону дат
- [ ] **Экспорт** — кнопка «Выгрузить CSV» для текущего фильтра

## Технические требования

- Маршрут: `/admin/payments`
- Защита: `roleGuard(UserRole.Admin)` из `@notary-portal/ui`
- RPC: `PaymentService.GetPaymentHistory`
- Переиспользовать существующий `TransactionTableComponent` из `@notary-portal/ui`
- Серверная пагинация (передавать page/limit в запрос)

## Acceptance criteria

- [ ] Таблица отображает все платежи системы с пагинацией
- [ ] Фильтры по статусу, типу и диапазону дат работают корректно
- [ ] Доступна выгрузка текущих результатов в CSV
- [ ] Клик на строку → переход к деталям платежа (маршрут из issue-17)
