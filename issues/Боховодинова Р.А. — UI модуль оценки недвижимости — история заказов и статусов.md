Реализовать страницу истории заказов оценки: лента с таймлайном изменений статусов, фильтры, повтор заказа, уведомления по изменениям.

## Ключевые функции

### `AssessmentHistoryComponent` — лента заказов

**Фильтры:**
- Период — select: «Всё время», «Последний месяц», «Последние 3 месяца», «Последний год»
- Статус — мультиселект из `AssessmentStatus`
- Поиск по адресу — text

**Лента `HistoryItemComponent`:**
- Карточка каждого заказа:
  - Адрес и тип объекта (заголовок)
  - Итоговая стоимость (если `Completed`/`Assessed`) — крупно
  - Статус — цветной бейдж
  - Дата создания и дата последнего обновления
  - **Мини-таймлайн** (горизонтальный) — 5 этапов с галочками/кружками по статусу
  - Кнопка «Подробнее» → `/applicant/assessment/results/:id`
  - Кнопка **«Повторить заказ»** — копирует параметры в новый `createAssessment` → переход к визарду

**Сортировка:** по дате (новые сверху по умолчанию, переключить на «Старые сверху»)

### `StatusTimelineComponent`

Горизонтальный таймлайн для встраивания в карточку:

```
[●]──[●]──[●]──[○]──[○]
Создан  Проверен  В работе  Оценён  Завершён
```

- `●` заполненный круг = статус пройден (зелёный)
- `●` с пульсацией = текущий статус (синий)
- `○` пустой = предстоящий статус (серый)
- CSS-анимация пульсации для текущего этапа (`.current { animation: pulse 2s infinite; }`)

**Входные параметры:**
- `@Input() currentStatus: string` — текущий статус из `AssessmentStatus` enum
- `@Input() compact: boolean = false` — компактный режим для карточки в ленте

### «Повторить заказ»
При нажатии:
1. Взять параметры из `assessment` (address, propertyType, description)
2. `createAssessment({ applicantId, ...params })`
3. Перейти в визард на шаг загрузки документов с новым `assessmentId`
