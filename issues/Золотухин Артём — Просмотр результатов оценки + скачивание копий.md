# Золотухин Артём — Просмотр результатов оценки и скачивание копий

## Контекст
- **Разделы:** `/applicant/assessment/results`, `/notary/assessment` (раздел результатов)  
- **Роли:** Заявитель / Нотариус  
- **Связанные файлы:** `libs/web/applicant/src/lib/`, `libs/web/notary/src/lib/`

---

## Задача
Реализовать страницу просмотра результатов оценки: карточка с итоговой стоимостью, детализация расчёта, список отчётов, скачивание PDF-копий.

---

## Маршруты

| Маршрут | Компонент | Описание |
|---|---|---|
| `/applicant/assessment/results` | `AssessmentResultsComponent` | Результаты оценки (список) |
| `/applicant/assessment/results/:assessmentId` | `AssessmentResultDetailComponent` | Детальная карточка |

## API-контракты (`@notary-portal/api-contracts`)

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `AssessmentService.listAssessments` | `{ filters: { userId, status: 'Assessed' или 'Completed' }, pagination }` | `{ assessments[], meta }` | Список завершённых оценок |
| `AssessmentService.getAssessment` | `{ id }` | `{ assessment }` | Детали оценки |
| `ReportService.listReports` | `{ assessmentId }` | `{ reports[] }` | Список отчётов |
| `ReportService.getReport` | `{ id }` | `{ report }` | Детали отчёта |

> **Важно:** `ReportService` **не экспортируется** из `@notary-portal/api-contracts` (см. `docs/api-contracts-by-module.md`).  
> Для блока отчётов использовать **mock-данные** с TODO-комментарием.

## Ключевые функции

### Список результатов (`/applicant/assessment/results`)
- Показывать только заявки со статусом `Assessed` или `Completed`
- Карточки: адрес, итоговая стоимость (`finalEstimatedValue`), дата оценки, кнопка «Подробнее»
- Пустой стейт: «Оценённых объектов пока нет»

### Детальная карточка (`/applicant/assessment/results/:assessmentId`)
- **Блок «Итог»:**
  - Оценочная стоимость крупным шрифтом: `finalEstimatedValue` отформатированный
  - Адрес, тип имущества, дата оценки
- **Блок «Детализация расчёта»:**
  - Mock-таблица: «Рыночная стоимость», «Корректировки», «Итоговая стоимость»
  - TODO для будущей реальной детализации
- **Блок «Отчёты»:**
  - Список `AssessmentReport` (mock): название, статус (бейдж), дата, кнопка «Скачать»
  - Кнопка «Скачать» (статус `Signed`): открывает `mockFileUrl` в новой вкладке
  - Незаверенные отчёты (статус `Draft`): кнопка disabled + подпись «Ожидает подписи»
- **Кнопка «Запросить копию»** → `/applicant/copies` (Имамов Д.Н.)
