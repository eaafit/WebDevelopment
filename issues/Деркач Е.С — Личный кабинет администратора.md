# Деркач Е.С. — Личный кабинет администратора: главная и управление заказами

## Контекст
- **Раздел:** `/admin`, `/admin/orders` — панель администратора  
- **Роли:** Администратор  
- **Связанные файлы:** `libs/web/admin/src/lib/`

---

## Задача
Реализовать главную страницу панели администратора (сводный дашборд) и страницу управления заявками.

---

## Маршруты

| Маршрут | Компонент | Описание |
|---|---|---|
| `/admin` | `AdminDashboardComponent` | Главная / сводный обзор |
| `/admin/orders` | `AdminOrdersComponent` | Управление всеми заявками |
| `/admin/orders/:id` | `AdminOrderDetailComponent` | Карточка заявки (модерация) |

---

## API-контракты (`@notary-portal/api-contracts`)

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `AssessmentService.listAssessments` | `{ pagination: { page:1, limit:100 } }` | `{ assessments[], meta: { totalItems } }` | Счётчик заявок на дашборде |
| `UserService.listUsers` | `{ pagination: { page:1, limit:1 } }` | `{ users[], meta: { totalItems } }` | Счётчик пользователей |
| `AssessmentService.listAssessments` | `{ pagination, filters }` | `{ assessments[], meta }` | Таблица заявок |
| `AssessmentService.getAssessment` | `{ id }` | `{ assessment }` | Карточка заявки |
| `AssessmentService.cancelAssessment` | `{ id, reason }` | `{ assessment }` | Отмена заявки |
| `AssessmentService.verifyAssessment` | `{ id, notaryId }` | `{ assessment }` | Ручное взятие в работу |

## Ключевые функции

### Дашборд (`/admin`)
- **Плитки метрик** (2×2 сетка):
  - Всего пользователей
  - Всего заявок
  - Заявок в работе (фильтр `InProgress`)
  - Завершённых за 30 дней
- **Таблица «Последние заявки»:** 5 записей, колонки — ID, адрес, статус, дата
- **Быстрые ссылки:** `/admin/users`, `/admin/orders`, `/admin/payments`, `/admin/monitoring`

### Управление заявками (`/admin/orders`)
- Таблица всех заявок: ID, адрес, userId заявителя, notaryId, статус, дата
- Фильтры: по статусу (`AssessmentStatus`), по дате создания
- Кнопки действий в строке: «Просмотр», «Отменить»
- Диалог отмены: поле «Причина» (обязательно)
- Пагинация

### Карточка заявки (`/admin/orders/:id`)
- Все поля заявки
- История смен статуса (если есть в данных)
- Кнопки: «Взять в работу» (ввод notaryId), «Отменить»
