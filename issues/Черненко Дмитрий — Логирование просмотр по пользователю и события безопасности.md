# Черненко Дмитрий (getpaintoo) — Логирование: просмотр по пользователю/заказу и события безопасности

## Контекст
- **Раздел:** `/admin/monitoring` — вкладки «По пользователям» и «События безопасности»  
- **Роли:** Администратор / частично Нотариус  
- **Связан с:** Нибылицын Лукьян (главная страница мониторинга, вкладка «Все события»)  
- **Связанные файлы:** `libs/web/admin/src/lib/`

---

## Задача
Реализовать две дополнительные вкладки на странице мониторинга: «По пользователям» (логи конкретного пользователя или заказа) и «События безопасности».

---

## Маршрут

Расширяет компонент `MonitoringComponent` (Нибылицын).  
Два компонента-вкладки подключаются как дочерние или через `@if`:

| Вкладка | Компонент | Путь |
|---|---|---|
| «По пользователям» | `AuditByEntityComponent` | `/admin/monitoring` → tab `by-entity` |
| «События безопасности» | `SecurityEventsComponent` | `/admin/monitoring` → tab `security` |

---

## Генерация компонентов

```bash
pnpm nx g @nx/angular:component libs/web/admin/src/lib/features/monitoring/audit-by-entity/audit-by-entity --standalone
pnpm nx g @nx/angular:component libs/web/admin/src/lib/features/monitoring/security-events/security-events --standalone
```

## Ключевые функции

### Вкладка «По пользователям» (`AuditByEntityComponent`)
- **Поле поиска** в шапке: ввод `userId` или `email` пользователя
- После ввода — фильтрация mock-данных по `actorId` или `actorEmail`
- Результат: хронологический список событий этого пользователя
- Альтернативный поиск по **заказу** — поле ввода `assessmentId`, фильтрация по `entityId` где `entityType === 'Assessment'`
- Переключатель: «По пользователю» / «По заказу»
- Таблица событий — те же колонки что в основной ленте (переиспользовать `AuditLogItemComponent` Нибылицына)

### Вкладка «События безопасности» (`SecurityEventsComponent`)
- Фильтр из mock-данных по полям:
  - `severity === 'warning' || 'error'`
  - Или специальные `action` типа: `'user.login_failed'`, `'user.blocked'`, `'token.revoked'`, `'permission.denied'`
- Таблица с акцентом на severity:
  - `warning` — жёлтый фон строки
  - `error` — красный фон строки
- **Счётчик** в заголовке вкладки: «[N] событий за 24ч»
- Кнопка **«Экспорт событий безопасности»** (CSV, только отфильтрованные строки)

## Интеграция с компонентом Нибылицына
Нибылицын создаёт `MonitoringComponent` и вкладку «Все события».  
Черненко встраивает свои компоненты в ту же страницу.

Переключение вкладок через `signal<'all' | 'by-entity' | 'security'>` currentTab в родительском `MonitoringComponent`:
```html
@switch (currentTab()) {
  @case ('all') { <app-audit-filters .../> <app-audit-list .../> }
  @case ('by-entity') { <app-audit-by-entity /> }
  @case ('security') { <app-security-events /> }
}
```

## Технические требования
- Импортировать mock-данные из `audit-logs.mock.ts` (файл Нибылицына)
- Вкладки — синхронизировать через `@Input() activeTab` или через общий signal
- Переиспользовать `AuditLogItemComponent` (файл Нибылицына) — не дублировать
