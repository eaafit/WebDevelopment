# Липовцев Родион — Администратор: создание, редактирование и удаление платежей

## Контекст
- **Раздел:** `/admin/payments/new`, `/admin/payments/:id/edit` — формы платежей  
- **Роли:** Администратор  
- **Связан с:** Трушин Евгений (список платежей) — его таблица вызывает эти компоненты  
- **Связанные файлы:** `libs/web/admin/src/lib/`

---

## Задача
Реализовать формы создания и редактирования платежа, а также модальное окно удаления.

---

## Маршруты

| Маршрут | Компонент | Описание |
|---|---|---|
| `/admin/payments/new` | `PaymentFormComponent` | Создание платежа |
| `/admin/payments/:id/edit` | `PaymentFormComponent` | Редактирование (тот же компонент) |

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `PaymentService.getPaymentHistory` | `{ pagination: { limit: 200 } }` | `{ payments[] }` | Загрузка платежа для режима редактирования |

> **Важно:** методы `createPayment`, `updatePayment`, `deletePayment` в proto **частично реализованы** (есть `createPayment`), но для редактирования/удаления — **mock**. Добавить TODO-комментарии.

| Метод | Запрос | Ответ | Где |
|---|---|---|---|

## Ключевые функции

### `PaymentFormComponent` — форма создания / редактирования
Определять режим по наличию `id` в маршруте (`ActivatedRoute.params`).

**Поля формы (`ReactiveFormsModule`):**

| Поле | Тип | Валидация |
|---|---|---|
| `userId` | text | required |
| `type` | select | required; значения из `PaymentType` enum |
| `amount` | number | required, > 0 |
| `currency` | select | required; варианты: RUB, USD, EUR |
| `status` | select | required; значения из `PaymentStatus` enum |
| `paymentDate` | datetime-local | required |
| `transactionId` | text | — |
| `paymentMethod` | text | — |
| `description` | textarea | — |
| `subscriptionId` | text | — |
| `assessmentId` | text | — |

**Режим создания:** все поля пустые, `status` = PENDING по умолчанию  
**Режим редактирования:** поля заполнены из существующего платежа (найти в списке по `id` через `getPaymentHistory`)

**Кнопки:**
- «Сохранить» → создание: `createPayment`, редактирование: mock  
- «Отмена» → `Router.back()` или `/admin/payments`

### `PaymentDeleteModalComponent`
- Принимает `@Input() payment: PaymentItem`
- Текст: «Вы уверены, что хотите удалить платёж **#[transactionId]**?»
- Кнопки: «Удалить» (деструктивный), «Отмена»
- `@Output() confirmed = new EventEmitter<void>()`
- `@Output() cancelled = new EventEmitter<void>()`
- После `confirmed` → mock-удаление → navigate `/admin/payments`

## Технические требования
- `PaymentType`, `PaymentStatus` enum из `@notary-portal/api-contracts` — использовать для select-опций
- При создании вызывать реальный `PaymentService.createPayment` через `RPC_TRANSPORT`
- При редактировании и удалении — mock + TODO-комментарий
- Добавить маршруты в `admin.routes.ts`:
  ```typescript
  { path: 'payments/new', component: PaymentFormComponent },
  { path: 'payments/:id/edit', component: PaymentFormComponent },
  ```
