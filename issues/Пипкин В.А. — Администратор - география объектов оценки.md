## Задача
Реализовать страницу с интерактивной картой (Leaflet) для визуализации географии заявок на оценку: маркеры по адресам/координатам, фильтры, переход к карточке заявки.

## API-контракты (`@notary-portal/api-contracts`)

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `AssessmentService.listAssessments` | `{ pagination: { page:1, limit:200 }, filters }` | `{ assessments[] }` | Маркеры на карте |

> В полях `Assessment` могут не быть координат — **геокодирование** адресов в lat/lng реализовать через бесплатный API Nominatim (`https://nominatim.openstreetmap.org/search?format=json&q=ADDRESS`). Если координаты недоступны — маркер не отображать.

## Ключевые функции

### Карта Leaflet
- Подключить Leaflet через `package.json` (`leaflet`, `@types/leaflet`) или CDN в `index.html`
- Отображать карту России по умолчанию, zoom 5
- Маркеры по каждой заявке, popup с данными:
  - Адрес
  - Статус (цветовой маркер: статус → цвет)
  - Дата создания
  - Ссылка «Открыть заявку» → `/admin/orders/:id`
- Цвета маркеров по статусу: `New` — серый, `InProgress` — синий, `Completed` — зелёный, `Cancelled` — красный

### Панель фильтров (над или рядом с картой)
- **Фильтр по статусу** — мультиселект
- **Фильтр по периоду** — два date-поля «от» и «до»
- Кнопка «Сбросить фильтры»
- При изменении фильтров — перезагрузить данные и перерисовать маркеры

### Геокодирование
- Для заявок без координат: запрос к Nominatim по полю `address`
- Кешировать результаты в `Map<string, { lat, lng }>` чтобы не дублировать запросы
- Rate limit Nominatim — не более 1 запроса/сек (добавить задержку через `sleep`)

## Фасад-сервис
Создать `libs/web/admin/src/lib/api/admin-geography-api.service.ts`:
```typescript
@Injectable({ providedIn: 'root' })
export class AdminGeographyApiService {
  private readonly client = createClient(AssessmentService, inject(RPC_TRANSPORT));
  listAssessmentsForMap(filters: MapFilters): Observable<Assessment[]>
  geocodeAddress(address: string): Promise<{ lat: number; lng: number } | null>
}
```

## Технические требования
- Leaflet инициализировать после `ngAfterViewInit` через `L.map(nativeElement)`
- Компонент `OnDestroy` — вызывать `map.remove()` для очистки
- Фильтры — `signal`-based или `FormGroup`
- Обновить `admin.routes.ts`

## Критерии готовности
- [ ] Карта Leaflet отображается на странице
- [ ] Маркеры загружаются из API и отображаются по координатам
- [ ] Геокодирование для заявок без координат работает
- [ ] Popup маркера со ссылкой на заявку
- [ ] Фильтры по статусу и периоду работают
- [ ] Responsive (карта занимает всю рабочую область)
