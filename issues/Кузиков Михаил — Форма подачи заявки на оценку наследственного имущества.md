- **Роли:** Заявитель (возможен Гость с последующей регистрацией)

## Задача
Реализовать многошаговую форму подачи заявки на оценку наследственного имущества: ввод данных объекта, выбор типа имущества, загрузка документов, согласия, отправка.

## API-контракты (`@notary-portal/api-contracts`)

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `AssessmentService.createAssessment` | `CreateAssessmentRequest { applicantId, propertyType, address, description, ... }` | `CreateAssessmentResponse { assessment }` | Отправка формы |
| `DocumentService.createDocument` | `CreateDocumentRequest { assessmentId, fileName, documentType, ... }` | `CreateDocumentResponse { document }` | Загрузка документа |
| `FormsService.saveAssessmentForm` | `SaveAssessmentFormRequest { assessmentId, formData }` | `SaveAssessmentFormResponse` | Сохранение формы |

> Эталон вызова RPC: `docs/frontend-api-contracts-guide.md`  
> `DocumentType` enum из `@notary-portal/api-contracts`: Passport, PropertyDeed, TechnicalPlan, CadastralPassport, Photo, Other

## Структура формы — 3 шага

### Шаг 1: Данные об объекте
- **Тип имущества** — select: «Квартира», «Дом», «Земельный участок», «Гараж», «Иное»
- **Адрес** — text с подсказками (Dadata API или просто text без интеграции на первом этапе)
- **Площадь** — number (кв. м), > 0
- **Кадастровый номер** — text (необязательно)
- **Описание** — textarea (необязательно)
- Кнопка «Далее»

### Шаг 2: Документы
- Загрузка файлов с указанием типа каждого:
  - Тип документа — select из `DocumentType` enum
  - `input[file]`: PDF / JPG / PNG, max 20 МБ каждый
  - Кнопка «Добавить документ» — добавляет новую строку
  - Кнопка удаления строки
- Минимум 1 документ (required)
- Кнопки «Назад», «Далее»

### Шаг 3: Подтверждение и согласия
- Сводка: тип имущества, адрес, список документов
- Чекбокс «Я подтверждаю достоверность данных» (required)
- Чекбокс «Согласен с условиями обработки персональных данных» (required)
- Ссылка «Политика конфиденциальности» (просто ссылка-заглушка)
- Кнопки «Назад», **«Отправить заявку»**

### Логика отправки
1. `AssessmentService.createAssessment({ applicantId: tokenStore.user().id, ... })` → получить `assessment.id`
2. Для каждого файла: `DocumentService.createDocument({ assessmentId, documentType, fileName })` (загрузка файлов — mock, реальный файловый upload не реализован в proto)
3. `FormsService.saveAssessmentForm({ assessmentId, formData: JSON.stringify(formValues) })`
4. Spinner «Отправка...» → success-экран → кнопка «К списку заявок» → `/applicant/orders`

## Технические требования
- Шаги через `signal<1 | 2 | 3>` currentStep
- Прогресс-бар или индикатор шагов в шапке формы
- Все данные между шагами хранить в родительском `NewOrderFormComponent`
- `ReactiveFormsModule` для каждого шага
- Обновить `applicant.routes.ts` — добавить `{ path: 'orders/new', component: NewOrderFormComponent }`
