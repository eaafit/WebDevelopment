## Задача
Реализовать главную страницу кабинета нотариуса и страницы управления заказами: список, фильтры, «взять в работу», смена статуса, документы по заказу.

| Маршрут | Компонент | Описание |

## API-контракты (`@notary-portal/api-contracts`)

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `AssessmentService.listAssessments` | `{ filters: { notaryId }, pagination }` | `{ assessments[], meta }` | Список заказов, дашборд |
| `AssessmentService.getAssessment` | `{ id }` | `{ assessment }` | Карточка заказа |
| `AssessmentService.verifyAssessment` | `{ id, notaryId }` | `{ assessment }` | Кнопка «Взять в работу» |
| `AssessmentService.completeAssessment` | `{ id, finalEstimatedValue }` | `{ assessment }` | Завершить заказ |
| `AssessmentService.cancelAssessment` | `{ id, reason }` | `{ assessment }` | Отменить заказ |
| `DocumentService.listDocuments` | `{ assessmentId }` | `{ documents[] }` | Документы по заказу |

> Эталон вызова RPC: `docs/frontend-api-contracts-guide.md`

## Ключевые функции

### Дашборд (`/notary`)
- Счётчики: «В работе», «Ожидают взятия», «Завершено за месяц»
- Последние 5 заказов (таблица)
- Быстрая навигация: Заказы, Подписка, Транзакции

### Список заказов (`/notary/orders`)
- Таблица: адрес, userId заявителя, статус (цветной бейдж), дата создания
- Фильтры: по статусу, по дате (диапазон)
- Кнопка **«Взять в работу»** — отображается только для заказов со статусом `New`; вызывает `verifyAssessment({ id, notaryId })`
- Пагинация
- Клик → `/notary/orders/:id`

### Карточка заказа (`/notary/orders/:id`)
- Детали заявки: адрес, тип, описание, статус, дата
- Блок документов: список через `DocumentService.listDocuments`
- Кнопка **«Завершить»** (видна при статусах `Verified`, `InProgress`):
  - Открывает диалог с полем «Итоговая оценочная стоимость» (число)
  - По подтверждению вызывает `completeAssessment({ id, finalEstimatedValue })`
- Кнопка **«Отменить»** (видна при статусах `New`, `Verified`, `InProgress`):
  - Открывает диалог с полем «Причина отмены» (текст, обязательно)
  - По подтверждению вызывает `cancelAssessment({ id, reason })`

## Фасад-сервис
Создать `libs/web/notary/src/lib/api/notary-assessments-api.service.ts`:

```typescript
@Injectable({ providedIn: 'root' })
export class NotaryAssessmentsApiService {
  private readonly client = createClient(AssessmentService, inject(RPC_TRANSPORT));
  // методы: listOrders(), getOrder(id), takeOrder(id), completeOrder(id, value), cancelOrder(id, reason)
}
```

> `notaryId` — `inject(TokenStore).user()?.id`

---

## Технические требования
- Диалоги подтверждения — standalone-компоненты с `@Output() confirmed`
- Статусы с цветами: `New` — серый, `Verified`/`InProgress` — синий, `Completed` — зелёный, `Cancelled` — красный
- После успешной смены статуса — перезагрузить данные (повторный запрос)
- Обновить `notary.routes.ts`
