# Иванова София — Личный кабинет заявителя

## Контекст
- **Раздел:** `/applicant` — кабинет заявителя  
- **Роли:** Заявитель  
- **Связанные файлы:** `libs/web/applicant/src/lib/`

---

## Задача
Реализовать главную страницу кабинета заявителя (дашборд) и страницы списка заявок и карточки заявки.

---

## Маршруты

| Маршрут | Компонент | Описание |
|---|---|---|
| `/applicant` | `ApplicantDashboardComponent` | Главная — обзор кабинета |
| `/applicant/orders` | `OrdersListComponent` | Список заявок |
| `/applicant/orders/:id` | `OrderDetailComponent` | Карточка заявки |

---

## API-контракты (`@notary-portal/api-contracts`)

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `AssessmentService.listAssessments` | `{ pagination: { page, limit }, filters: { userId } }` | `{ assessments[], meta }` | Список заявок, дашборд |
| `AssessmentService.getAssessment` | `{ id: string }` | `{ assessment }` | Карточка заявки |
| `NotificationService.listNotifications` | `{ userId, pagination }` | `{ notifications[], meta }` | Дашборд — последние события |

> **Важно:** `NotificationService` **не экспортируется** из `@notary-portal/api-contracts` на данный момент (см. `docs/api-contracts-by-module.md`). Для блока уведомлений использовать **mock-данные** с TODO-комментарием.  
> Эталон вызова RPC: `docs/frontend-api-contracts-guide.md`

## Ключевые функции

### Дашборд (`/applicant`)
- **Виджет «Мои заявки»:** счётчики по статусам — «Новые», «В работе», «Завершённые» (из `AssessmentStatus` enum)
- **Виджет «Последние события»:** 5 последних mock-уведомлений (когда NotificationService не доступен)
- **Кнопка «Подать заявку»** → `/applicant/orders/new`
- **Быстрые ссылки:** Документы, Уведомления, Поддержка

### Список заявок (`/applicant/orders`)
- Таблица или карточки: адрес, тип имущества, статус (цветной бейдж), дата создания
- Фильтр по статусу (`AssessmentStatus` enum из `@notary-portal/api-contracts`)
- Пагинация
- Клик по строке → `/applicant/orders/:id`

### Карточка заявки (`/applicant/orders/:id`)
- Заголовок: адрес и статус
- Детали: описание, тип имущества, дата создания, оценочная стоимость
- Блок «Документы»: список прикреплённых документов (только отображение)
- Блок «Комментарии»: статичное отображение (форма добавления — заглушка)

## Фасад-сервис

Создать `libs/web/applicant/src/lib/api/assessments-api.service.ts`:

```typescript
@Injectable({ providedIn: 'root' })
export class AssessmentsApiService {
  private readonly client = createClient(AssessmentService, inject(RPC_TRANSPORT));
  // методы: listAssessments(), getAssessment(id)
}
```

> `RPC_TRANSPORT` импортировать из `@notary-portal/ui`  
> `userId` брать из `inject(TokenStore).user()?.id`

---

## Технические требования
- Все RPC-вызовы только через фасад-сервис, **не в компонентах**
- `AssessmentStatus` отображать по-русски: `New` → «Новая», `Verified` → «Проверена», `InProgress` → «В работе», `Assessed` → «Оценена», `Completed` → «Завершена», `Cancelled` → «Отменена»
- Состояния: загрузка (skeleton/spinner), пустой список, ошибка
- Обновить `applicant.routes.ts` — добавить маршруты orders и orders/:id
