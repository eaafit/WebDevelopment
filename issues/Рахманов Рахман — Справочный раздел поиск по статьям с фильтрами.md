## Задача
Реализовать компонент расширенного поиска по статьям FAQ с фильтрами по автору и логике поиска (OR / AND / left / right / contains / equal).

---

## Маршрут
Поиск встраивается в `FaqPageComponent` (Титов) через `<lib-faq-search>`.  
Query-параметр `?q=` активирует режим поиска.

## Ключевые функции

### `FaqSearchComponent` — строка поиска с расширенными опциями

**Базовая строка поиска:**
- `input[text]` с иконкой лупы
- Поиск при вводе (debounce 300 мс через `debounceTime` из RxJS)
- Подсветка совпадений в результатах
- Кнопка «Расширенный поиск» → разворачивает панель опций

**Панель расширенного поиска:**

| Параметр | UI-элемент | Варианты |
|---|---|---|
| **Логика поиска** | Radio-группа | OR, AND, «Слева» (left), «Справа» (right), «Содержит» (contains), «Точное совпадение» (equal) |
| **Поиск по полям** | Checkboxes | Заголовок, Содержание, Теги |
| **Автор** | Select | «Все авторы» + список уникальных авторов из mock |
| **Категория** | Select | «Все категории» + список из mock |
| **Диапазон дат** | Два `input[date]` | Дата публикации от/до |
| **Сортировка** | Select | «По релевантности», «По дате», «По просмотрам» |

**Описание логик поиска (реализовать в `search.utils.ts`):**
- `OR` — статья содержит хотя бы одно слово из запроса
- `AND` — статья содержит все слова запроса
- `left` — текст статьи начинается с запроса (`startsWith`)
- `right` — текст статьи заканчивается на запрос (`endsWith`)
- `contains` — запрос входит в текст подстрокой (`includes`)
- `equal` — точное совпадение заголовка с запросом

---

### `FaqSearchResultsComponent` — результаты

- Счётчик: «Найдено [N] статей»
- Список `FaqArticlePreviewComponent`:
  - Заголовок статьи с **подсветкой совпадений** (`<mark>`)
  - Категория (иконка + название)
  - Автор, дата публикации
  - Фрагмент текста с подсветкой (первые 150 символов содержащего совпадение абзаца)
  - Теги (matching теги выделены)
- Пустой стейт: «Ничего не найдено. Попробуйте изменить запрос.»
- Сохранение последних 5 запросов в `localStorage` → «Недавние поиски»

## Технические требования
- Debounce через `Subject` + `debounceTime(300)` из RxJS
- Подсветка — `DomSanitizer.bypassSecurityTrustHtml` для innerHTML
- Последние поиски — `localStorage` (допустимо здесь, т.к. это пользовательские настройки UI)
- `FaqSearchComponent` принимает `@Input() initialQuery?: string` (из query-param `?q=`)
- `@Output() resultsChange: EventEmitter<FaqArticle[]>` — передаёт результаты наверх
