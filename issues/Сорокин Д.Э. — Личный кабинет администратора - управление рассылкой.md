# Сорокин Д.Э. — Администратор: просмотр списка рассылок

## Контекст
- **Раздел:** `/admin/newsletter` — управление рассылками  
- **Роли:** Администратор  
- **Связан с:** Карлов И.А. (создание рассылки) — кнопка «Создать» ведёт на его компонент  
- **Связанные файлы:** `libs/web/admin/src/lib/`

---

## Задача
Реализовать страницу списка рассылок с фильтрацией по статусу и боковой панелью деталей.

---

## Маршрут

| Маршрут | Компонент | Описание |
|---|---|---|
| `/admin/newsletter` | `NewsletterListComponent` | Список рассылок |

---

## Генерация компонентов

```bash
pnpm nx g @nx/angular:component libs/web/admin/src/lib/features/newsletter/newsletter-list/newsletter-list --standalone
pnpm nx g @nx/angular:component libs/web/admin/src/lib/features/newsletter/newsletter-detail-panel/newsletter-detail-panel --standalone
```

---

## API-контракты
Методы рассылок в proto **не реализованы**. Использовать **mock-данные**.

---

## Ключевые функции

### Список рассылок (`/admin/newsletter`)
- Кнопка **«Создать рассылку»** → `/admin/newsletter/new` (компонент Карлова И.А.)
- Таблица рассылок:
  - Тема письма
  - Дата отправки
  - Количество получателей
  - Статус: `sent` «Отправлено», `draft` «Черновик», `scheduled` «Запланировано»
- Фильтр по статусу — select
- Mock-данные: 10–12 записей в `newsletter.mock.ts`
- Клик по строке → открывает `NewsletterDetailPanelComponent` (боковая панель)

### `NewsletterDetailPanelComponent`
- Заголовок: тема рассылки
- Поля: дата, получатели (количество + список email первых 5)
- Превью текста (первые 200 символов)
- Кнопка «Отправить повторно» (mock — success toast)
- Кнопка «Закрыть» (скрыть панель)
