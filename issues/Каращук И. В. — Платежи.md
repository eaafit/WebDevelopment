# Каращук И.В. — Платежи: выбор тарифа, оплата, история

## Контекст
- **Разделы:** `/applicant/payments`, `/notary/subscription`, `/notary/subscription/checkout`  
- **Роли:** Нотариус / Заявитель  
- **Связанные файлы:** `libs/web/applicant/src/lib/`, `libs/web/notary/src/lib/`

---

## Задача
Реализовать полный платёжный флоу: история платежей, выбор тарифа подписки, оформление оплаты с промокодом.

## API-контракты (`@notary-portal/api-contracts`)

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `PaymentService.getPaymentHistory` | `{ pagination, filters: { userId } }` | `{ payments[], meta }` | История платежей |
| `PaymentService.getSubscription` | `{ userId }` | `{ subscription }` | Текущая подписка |
| `PaymentService.createSubscription` | `{ userId, plan }` | `{ subscription }` | Оформление подписки |
| `PaymentService.createPayment` | `{ userId, type, amount, currency, description }` | `{ payment }` | Создание платежа |

> Эталон маппинга `getPaymentHistory`: `libs/web/notary/src/lib/features/dashboard/transactions/transactions-api.service.ts`  
> `PromoService.validatePromo` — **не экспортируется**, промокод валидировать через mock (любой код ≥ 6 символов → скидка 10%)

## Ключевые функции

### История платежей (`/applicant/payments`)
- Таблица: тип, сумма, статус (бейдж), дата
- Фильтры: статус, тип, даты
- Пагинация
- Переиспользовать компонент `TransactionTable` из `@notary-portal/ui` если возможно

### Страница подписки нотариуса (`/notary/subscription`)
- Блок «Текущая подписка»: план, дата окончания, статус (активна/истекла)
- Если нет подписки → промо-блок «Подключите подписку»
- Сетка карточек планов: **Basic / Premium / Enterprise**
  - `SubscriptionPlan` enum из `@notary-portal/api-contracts`
  - Mock-данные для цен и описаний (в `subscription-plans.mock.ts`)
  - Кнопка «Выбрать» → `/notary/subscription/checkout?plan=basic`

### Оформление подписки (`/notary/subscription/checkout`)
**Многошаговый флоу:**

**Шаг 1 — Сводка**
- Выбранный план, цена, дата начала
- Кнопка «Далее»

**Шаг 2 — Промокод**
- Поле ввода промокода (опционально)
- Кнопка «Применить» → mock-валидация → если ≥ 6 символов: «Скидка 10% применена»
- Итоговая сумма с учётом скидки
- Кнопки «Назад», «Далее»

**Шаг 3 — Способ оплаты**
- Радиогруппа: «Банковская карта», «СБП», «Баланс аккаунта»
- Визуальный выбор (иконки)
- При «Банковская карта» — mock-форма (поля карты, disabled, подпись «Тестовый режим»)
- Кнопки «Назад», «Оплатить»

**Шаг 4 — Success**
- Иконка ✓, «Подписка оформлена!»
- Детали: план, сумма, дата окончания
- Кнопка «В кабинет»
- Вызов `createSubscription` при подтверждении на шаге 3

## Технические требования
- Шаги через `signal<number>` currentStep или `enum Step`
- `userId` из `inject(TokenStore).user()?.id`
- `SubscriptionPlan` enum из `@notary-portal/api-contracts`
- Суммы: `Intl.NumberFormat('ru-RU', { style: 'currency', currency: 'RUB' })`
- Обновить `applicant.routes.ts` и `notary.routes.ts`
