# Говор Сергей — Чат поддержки: интерфейс оператора (admin-сторона)

## Контекст
- **Роль поддержки:** Администратор (оператор) — отвечает на тикеты пользователей  
- **Связан с:** Васорин Иван (пользовательская сторона чата) — работают с теми же тикетами  
- **Связанные файлы:** `libs/web/admin/src/lib/`

---

## Задача
Реализовать интерфейс оператора поддержки: очередь тикетов, смена статуса, ответы от имени поддержки, SLA-мониторинг всех тикетов.

## Ключевые функции

### `AdminSupportComponent` — трёхколоночный layout
- **Левая (узкая):** панель фильтров и сортировки
- **Центральная:** `TicketQueueComponent` — список всех тикетов
- **Правая:** `OperatorChatComponent` — активный тикет

### `TicketQueueComponent` (операторская очередь)
Отличия от пользовательского `TicketListComponent`:
- Показывает **все** тикеты всех пользователей, не только свои
- Дополнительные колонки: имя/email пользователя, дата создания
- **Фильтры:**
  - Статус: `open` / `in_progress` / `resolved` / `closed`
  - SLA: «Нарушен» / «Скоро нарушится (< 2ч)» / «В норме»
  - Сортировка: по дате, по SLA-дедлайну
- SLA-бейдж на каждом тикете (цветовая система из задания Васорина)
- Кнопка **«Взять в работу»** для тикетов со статусом `open` → меняет статус на `in_progress`

### `OperatorChatComponent`
- То же отображение переписки что у Васорина
- Сообщения оператора отмечаются как `authorRole: 'support'`
- **Дополнительно:**
  - Кнопки смены статуса тикета: «Решено», «Закрыть» (с диалогом подтверждения)
  - Блок «Информация о пользователе»: email, роль, дата регистрации

### SLA-монитор (блок в шапке страницы)
- 3 счётчика: «Открытых», «Нарушено SLA», «Решено сегодня»
- Цвета: нарушено — красный, скоро нарушится — жёлтый
