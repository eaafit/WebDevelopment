# Мирошник И.О. — Уведомления: лента, фильтры, настройки каналов

## Контекст
- **Разделы:** `/applicant/notifications`, `/notary/notifications`, `/admin/notifications`  
- **Роли:** Заявитель / Нотариус / Администратор  
- **Связанные файлы:** `libs/web/applicant/`, `libs/web/notary/`, `libs/web/admin/`

## Задача
Реализовать страницу уведомлений: лента с фильтрацией по статусу прочитанности, кнопки «прочитать всё», удаление, история событий, настройки каналов доставки.

## Ключевые функции

### Лента уведомлений
- **Счётчик непрочитанных** в заголовке страницы
- **Кнопка «Прочитать все»** → mock-вызов `markAllAsRead` → все становятся прочитанными
- **Фильтр**: «Все» / «Непрочитанные» / «Прочитанные»
- Список `NotificationItemComponent` для каждого уведомления:
  - Иконка типа (статус оценки / платёж / документ / система)
  - Заголовок + текст (truncate при > 2 строк)
  - Дата и время
  - Полужирный текст если не прочитано
  - Кнопка «✓ Прочитано» — mock `markAsRead`
  - Кнопка «✕ Удалить» — mock `deleteNotification`
- Пагинация или «Загрузить ещё»

### Настройки каналов (`NotificationSettingsComponent`)
- Блок «Настройки уведомлений» в нижней части страницы (или отдельная вкладка)
- Тогглы (checkbox/toggle switch):
  - Email-уведомления
  - Push-уведомления в браузере
  - SMS-уведомления
- Для каждого типа событий: изменение статуса заявки, платёж, документ, системные
- Сохранение через mock с success-toast

## Технические требования
- Состояние уведомлений: `signal<NotificationItem[]>` в компоненте
- Операции (markAsRead, delete) — обновлять локальный `signal` оптимистично
- Счётчик непрочитанных пересчитывать через `computed()`
- Экспортировать компоненты из `@notary-portal/ui`
- Подключить компоненты в маршруты `applicant.routes.ts`, `notary.routes.ts`, `admin.routes.ts`
