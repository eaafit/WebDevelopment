## API-контракты (`@notary-portal/api-contracts`)

| Метод | Запрос | Ответ |
|---|---|---|
| `AuthService.login` | `{ email: string, password: string }` | `LoginResponse { result: { accessToken, refreshToken, user } }` |
| `AuthService.register` | `{ email, password, fullName, phoneNumber, role: number }` | `RegisterResponse { result: { accessToken, refreshToken, user } }` |

> Сохранение токенов и редирект по роли выполняет **`AuthService`** из `@notary-portal/guest` — вызывай `authService.login(email, password)` и `authService.register(params)`.  
> Эталон вызова RPC: `libs/web/notary/src/lib/features/dashboard/transactions/transactions-api.service.ts`  
> Подробнее: `docs/frontend-api-contracts-guide.md`

## Ключевые функции

### Форма входа (`/auth`) — доработать существующий `login.ts`
- Поля: `email`, `password` (уже сделано в предыдущей итерации — проверь что они есть)
- Кнопка «Войти» — вызывает `authService.login(email, password)`
- Ссылки: «Регистрация» → `/auth/register`, «Забыли пароль?» → `/auth/forgot-password`
- Отображение ошибки из `authService.error()` сигнала
- Блокировка кнопки при `authService.loading()`

### Форма регистрации (`/auth/register`)
- Поля: `fullName`, `email`, `phoneNumber`, `password`, `confirmPassword`
- Выбор роли: радиогруппа **«Заявитель» / «Нотариус»** (значения `1` и `2` — см. `UserRole` в `@notary-portal/ui`)
- Чекбокс согласия с условиями (required)
- Валидация: все поля обязательны, email-формат, пароль ≥ 8 символов, `password === confirmPassword`
- Вызов: `authService.register({ fullName, email, phoneNumber, password, role })`

### Форма восстановления пароля (`/auth/forgot-password`)
- Поле: `email`
- Кнопка «Отправить ссылку»
- Метода в API нет — реализовать success-заглушку: 1.5 сек «загрузка» → «Письмо отправлено»
- Ссылка «Назад ко входу»

## Технические требования
- Использовать `AuthService` из `@notary-portal/guest` — **не создавать новый RPC-клиент в компоненте**
- `FormsModule` или `ReactiveFormsModule` — на выбор
- Показывать состояние загрузки через `authService.loading()` сигнал
- Ошибки: отображать под формой через `authService.error()`
- Responsive layout (mobile-first)
