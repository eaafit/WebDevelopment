## API-контракты (`@notary-portal/api-contracts`)

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `AssessmentService.getAssessment` | `{ id: assessmentId }` | `{ assessment }` | Загрузка параметров для сводки |

## Ключевые функции

### `AssessmentRequestFormComponent`

**Блок «Сводка параметров»** (read-only):
- Данные из `getAssessment`: адрес, тип объекта, площадь, кадастровый номер
- Количество загруженных документов (из `listDocuments`)
- Кнопка «Изменить параметры» → переход на шаг 1

**Блок «Параметры запроса»**:
- **Срочность** — radio-группа:
  - «Стандарт (5–7 рабочих дней)» — бесплатно
  - «Ускоренная (2–3 дня)» — +50% к стоимости (mock)
  - «Срочная (1 день)» — +100% к стоимости (mock)
- **Комментарий к запросу** — textarea, необязательно

**Блок «Предварительная стоимость»** (mock):
- Базовая стоимость: рассчитать из типа объекта и площади по mock-формуле
- С учётом срочности: базовая × коэффициент
- Форматировать через `Intl.NumberFormat`

**Кнопки:**
- «← Назад» — шаг 2
- **«Отправить запрос»** — вызов `verifyAssessment({ id: assessmentId, notaryId: null })` → переход к экрану отслеживания

---

### `AssessmentProgressTrackerComponent`

Показывает после успешной отправки:

**Визуальный таймлайн этапов:**
```
✅ Заявка создана
✅ Документы загружены  
🔄 Назначение оценщика  (текущий)
⭕ Оценка в процессе
⭕ Результат готов
```

- Иконка текущего этапа — анимированный spinner
- Завершённые этапы — зелёная галочка
- Будущие этапы — серые точки
- Примерная дата завершения (mock: сейчас + N дней по срочности)

**Кнопки:**
- «К списку заявок» → `/applicant/orders`
- «Статус оценки» → `/applicant/assessment/history` (Боховодинова)

---

## Технические требования
- `@Input() assessmentId: string` — принимает из `AssessmentWizardService` (Бурцева)
- Этапы в `ProgressTrackerComponent` — определяются по `AssessmentStatus` enum:
  - `New` → «Заявка создана» завершён, «Назначение оценщика» — текущий
  - `Verified` → «Назначение оценщика» завершён, «Оценка в процессе» — текущий
  - `Assessed` → «Оценка в процессе» завершён, «Результат готов» — текущий
  - `Completed` → все этапы завершены
