# Емельянов Д.А. — Модуль оценки: загрузка фото и документов объекта

## Контекст
- **Разделы:** `/applicant/assessment`, `/notary/assessment`  
- **Роли:** Заявитель / Нотариус  
- **Связан с:** Бурцева Мария (ввод параметров объекта) — шаг «Загрузка» идёт после или вместе с вводом параметров  
- **Связанные файлы:** `libs/web/shared/ui/src/lib/`

---

## Задача
Реализовать шаг «Загрузка фото и документов» в модуле оценки: массовая загрузка, сортировка, предпросмотр, метки типа фото, требования к качеству.

---

## Маршрут

Компонент встраивается как **шаг** в многошаговую форму модуля оценки `/applicant/assessment`:

| Маршрут | Компонент | Описание |
|---|---|---|
| `/applicant/assessment` | `AssessmentWizardComponent` | Многошаговый визард |
| — шаг «Загрузка» | `AssessmentUploadStepComponent` | Этот компонент |

---

## Генерация компонентов

```bash
pnpm nx g @nx/angular:component libs/web/shared/ui/src/lib/assessment/upload-step/upload-step --standalone
pnpm nx g @nx/angular:component libs/web/shared/ui/src/lib/assessment/photo-item/photo-item --standalone
```

Экспортировать из `@notary-portal/ui`.

## API-контракты (`@notary-portal/api-contracts`)

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `DocumentService.createDocument` | `{ assessmentId, fileName, documentType, fileSize }` | `{ document }` | Создание записи файла |
| `DocumentService.deleteDocument` | `{ id }` | `{ success }` | Удаление |

> Реального upload-а нет: `createDocument` создаёт только метаданные. Предпросмотр — из `URL.createObjectURL(file)`.  
> `DocumentType` enum: `Photo`, `TechnicalPlan`, `CadastralPassport`, `PropertyDeed`, `Passport`, `Other`

## Ключевые функции

### `AssessmentUploadStepComponent`

**Зона загрузки:**
- Drag&Drop + клик (переиспользовать или расширить `FileUploadZoneComponent` Головатого)
- Принимает: JPG, JPEG, PNG (фото) + PDF (документы)
- Max 20 МБ/файл, max 30 файлов всего
- Счётчик: «[N] / 30 файлов»

**Сетка превью фотографий:**
- Thumbnail-сетка (3–4 колонки)
- Каждый элемент `PhotoItemComponent`:
  - Превью (img / PDF-иконка)
  - **Метка типа** — select:
    - «Фасад объекта», «План помещения», «Правоустанавливающий документ», «Кадастровый паспорт», «Паспорт», «Прочее»
    - Соответствует `DocumentType` enum
  - Индикатор качества (только для фото): зелёный/жёлтый/красный
    - Зелёный: > 1 МБ и разрешение > 1024px (mock-проверка по размеру файла)
    - Жёлтый: 200 КБ – 1 МБ
    - Красный: < 200 КБ
  - Кнопка удаления
  - Drag-handle для изменения порядка

**Drag-сортировка:**
- Перетаскивание элементов для изменения порядка
- Реализовать через `HTML5 Drag and Drop API`: `dragstart`, `dragover`, `drop`
- Визуальный placeholder при перетаскивании

**Блок «Требования к фото»** (информационный, складывается):
- Список требований: «Чёткое изображение», «Не менее 1 МБ», «Съёмка при хорошем освещении», и т.д.

**Кнопки:**
- `@Input() assessmentId: string`
- `@Output() uploadComplete: EventEmitter<void>` — все файлы отмечены и записи созданы

## Технические требования
- Качество фото: проверять `File.size` — без реального декодирования изображения
- Состояние: `signal<UploadedFile[]>` где `UploadedFile = { file: File; type: DocumentType; documentId?: string; quality: 'good' | 'ok' | 'low' }`
- Создавать запись `DocumentService.createDocument` для каждого добавленного файла
- `DragEvent` — обработчики с `preventDefault()` для `dragover`
