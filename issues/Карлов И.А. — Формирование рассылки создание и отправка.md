## API-контракты
Методы рассылок в proto **не реализованы**. Форма работает на mock с имитацией отправки.  
Для выбора получателей вручную — использовать `UserService.listUsers` для подгрузки списка.

| Метод | Запрос | Ответ | Где |
|---|---|---|---|
| `UserService.listUsers` | `{ pagination: { limit: 100 } }` | `{ users[] }` | Поиск получателей вручную |

## Ключевые функции

### Layout страницы
- **Левая колонка (2/3):** форма
- **Правая колонка (1/3):** превью письма (обновляется при вводе)

### Форма (`ReactiveFormsModule`)
- **Тема письма** — `input[text]`, обязательное поле
- **Содержимое** — `textarea` с базовыми кнопками форматирования:
  - Жирный (оборачивает выделение в `<strong>`)
  - Курсив (`<em>`)
  - Список (`<ul><li>`)
  - Использовать `contenteditable div` или простые кнопки с `document.execCommand`
- **Аудитория** — радиогруппа:
  - «Все пользователи»
  - «Только нотариусы» (UserRole Notary)
  - «Только заявители» (UserRole Applicant)
  - «Выбрать вручную» → показывает поле поиска
- **Поиск получателей вручную** (условный блок):
  - Поле ввода email — фильтрует список из `UserService.listUsers`
  - Добавленные адреса показываются как теги с кнопкой удаления
- **Дата отправки** — радио: «Сейчас» / «Запланировать»
  - При «Запланировать» — `input[datetime-local]`
- **Кнопки:**
  - «Сохранить как черновик» (mock)
  - «Отправить» → диалог подтверждения → mock-отправка (1.5 сек spinner) → success-toast → redirect на `/admin/newsletter`

### Превью письма
- Обновляется в реальном времени при изменении темы и содержимого
- Показывает стилизованный email-макет: шапка, тема, тело
