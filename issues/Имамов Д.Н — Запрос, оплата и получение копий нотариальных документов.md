# Имамов Д.Н. — Запрос, оплата и получение копий нотариальных документов

## Контекст
- **Разделы:** `/applicant/copies`, `/notary/copies`  
- **Роли:** Заявитель / Нотариус  
- **Связанные файлы:** `libs/web/applicant/src/lib/`, `libs/web/notary/src/lib/`

---

## Задача
Реализовать полный флоу запроса копии нотариального документа: форма запроса, прикрепление оснований, расчёт стоимости, оплата, статус выдачи, скачивание.

## API-контракты (`@notary-portal/api-contracts`)

> **Важно:** `SaleService` (методы для запросов на копии) **не экспортируется** из `@notary-portal/api-contracts` (см. `docs/api-contracts-by-module.md`).  
> Использовать **mock-данные** с TODO-комментарием. `PaymentService.createPayment` — реальный вызов при оплате.

| Метод | Запрос | Реальный? | Где |
|---|---|---|---|
| `SaleService.createSale` | `{ userId, documentType, ... }` | ❌ mock | Создание запроса |
| `SaleService.listSales` | `{ userId, pagination }` | ❌ mock | Список запросов |
| `PaymentService.createPayment` | `{ userId, type: DOCUMENT_COPY, amount, ... }` | ✅ реальный | Оплата |

## Mock-структура запроса копии
```typescript
export interface CopyRequest {
  id: string;
  userId: string;
  documentType: 'assessment_report' | 'notarial_deed' | 'extract';
  status: 'pending' | 'processing' | 'ready' | 'delivered';
  createdAt: string;
  price: number;
  currency: string;
  downloadUrl: string | null;
  isPaid: boolean;
}
```

## Ключевые функции

### Страница копий (`/applicant/copies`)

**Блок «Мои запросы»:**
- Список `CopyStatusCardComponent` для каждого запроса:
  - Тип документа (читаемо)
  - Статус с иконкой: «Ожидает оплаты», «В обработке», «Готово», «Выдано»
  - Дата запроса
  - Кнопка **«Скачать»** (если статус `ready` / `delivered` и есть `downloadUrl`)
  - Кнопка **«Оплатить»** (если `!isPaid`) → открывает платёжный флоу
- Пустой стейт: «Нет запросов» + кнопка «Подать запрос»

**Форма запроса (`CopyRequestFormComponent`):**
- Открывается кнопкой «Подать запрос» (inline или боковая панель)
- Поля:
  - Тип документа — select: «Отчёт об оценке», «Нотариальный акт», «Выписка»
  - Номер заявки / ID документа — text
  - Основание запроса — textarea (обязательно)
  - Прикрепить документ-основание — `input[file]`, PDF/JPG/PNG, max 10 МБ
- Расчёт стоимости: mock — фиксированная цена по типу документа
- Кнопка «Оформить запрос» → mock `createSale` → добавить в список → перейти к оплате

**Платёжный флоу (упрощённый):**
- Диалог: «Оплатить [сумма] ₽ за копию [тип]?»
- Кнопка «Оплатить» → `PaymentService.createPayment({ type: DOCUMENT_COPY, amount, userId })`
- Success → обновить статус карточки на «В обработке»

## Технические требования
- `PaymentType.DOCUMENT_COPY` из `@notary-portal/api-contracts` — использовать при создании платежа
- Цены в mock: «Отчёт об оценке» = 500 ₽, «Нотариальный акт» = 300 ₽, «Выписка» = 150 ₽
- Статусы с цветами: `pending` — жёлтый, `processing` — синий, `ready` — зелёный, `delivered` — серый
- Экспортировать компоненты из `@notary-portal/ui`
- Обновить `applicant.routes.ts` и `notary.routes.ts`
