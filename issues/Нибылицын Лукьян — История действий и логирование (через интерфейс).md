## Mock-структура события
```typescript
export interface AuditLog {
  id: string;
  actorId: string;       // кто совершил действие
  actorEmail: string;
  actorRole: string;
  action: string;        // 'assessment.created' | 'user.blocked' | 'payment.completed' | ...
  entityType: string;    // 'Assessment' | 'User' | 'Payment' | ...
  entityId: string;
  description: string;   // «Заявка #123 создана»
  ipAddress: string;
  createdAt: string;     // ISO-datetime
  severity: 'info' | 'warning' | 'error';
}
```
Mock-данные: 50 записей в `audit-logs.mock.ts` с разными типами событий.

## Ключевые функции

### Страница мониторинга (`/admin/monitoring`)
- Заголовок с кнопкой **«Экспорт CSV»**
- Ссылки-вкладки: «Все события» (Лукьян) | «По пользователям» (Черненко) | «События безопасности» (Черненко)

### `AuditFiltersComponent`
- Фильтры:
  - Тип действия — select (из уникальных `action` в mock-данных)
  - Тип сущности — select (из уникальных `entityType`)
  - Severity — select: «Все», «Info», «Warning», «Error»
  - Дата «от» / «до» — два `input[date]`
  - actorEmail — поле поиска
- Кнопка «Сбросить»
- `@Output() filtersChanged: EventEmitter<AuditFilters>`

### Лента событий
- Список `AuditLogItemComponent`:
  - Цветная иконка по severity (info — синий, warning — жёлтый, error — красный)
  - `createdAt` — «ДД.ММ.ГГГГ ЧЧ:ММ:СС»
  - actorEmail + actorRole
  - Описание действия
  - entityType + entityId
  - IP-адрес (серый)
- Клик по строке → открывает детальную панель (inline expand или боковая)
- Пагинация (клиентская по mock-данным)

### Экспорт CSV
```typescript
const csv = filteredLogs.map(l => 
  [l.createdAt, l.actorEmail, l.action, l.entityType, l.entityId, l.ipAddress, l.severity].join(',')
).join('\n');
```
