# Матрица регрессии админских платежей

Документ фиксирует проверки после изменений в `/admin/payments`, `/admin/payments/new`, `/admin/orders` и связанных платежных сценариях. Он нужен как короткая памятка для учебного проекта: раздел платежей связан с CSV, чеками, audit, Robokassa, уведомлениями и заявками, поэтому небольшая правка в UI легко может проявиться в другой части админки.

## Быстрый smoke

| Шаг | Страница | Действие | Ожидание |
|---|---|---|---|
| 1 | `/admin/payments` | Открыть без фильтров | Нет бесконечной загрузки |
| 2 | `/admin/payments` | Сменить страницу | Таблица показывает другую выборку |
| 3 | `/admin/payments` | Сменить `На странице` | Запрос уходит с новым limit |
| 4 | `/admin/payments` | Открыть `Все статусы` | Dropdown поверх таблицы |
| 5 | `/admin/payments` | Нажать Escape в dropdown | Dropdown закрыт |
| 6 | `/admin/payments` | Экспорт CSV | Файл скачивается |
| 7 | `/admin/payments` | Экспорт пустой выборки | Ошибка видна в toolbar |
| 8 | `/admin/payments` | Открыть детали | Модалка не ломает фон |
| 9 | `/admin/payments` | Открыть чек | Новая вкладка или понятная ошибка |
| 10 | `/admin/payments` | Скачать чек | Файл или понятная ошибка |
| 11 | `/admin/payments/new` | Оставить обязательные поля пустыми | Сетка формы не плывет |
| 12 | `/admin/orders` | Открыть заявки | Raw `internal error` не показывается |
| 13 | `/admin/orders` | Открыть фильтр нотариуса | Dropdown поверх таблицы |
| 14 | `/admin/orders` | Завершить заявку при ошибке API | Кнопка снова доступна |
| 15 | `/admin/monitoring` | Открыть audit list | Ошибка нормализована |

## Пагинация и таблица платежей

| Проверка | Данные | Ожидание |
|---|---|---|
| Первая загрузка | `page=1`, `limit=10` | `loading=false`, строки показаны |
| Переход вперед | `page=2` | Запрос отличается от первой страницы |
| Переход назад | `page=1` | Возврат к первой странице |
| Смена лимита | `10 -> 20` | `currentPage=1` |
| Смена лимита | `20 -> 50` | `limit=50` |
| Общий счетчик | `totalItems=348` | UI показывает 348 |
| Узкий ID | Длинный UUID | Колонка не растягивает всю таблицу |
| Resize ID | Drag вправо | Ширина увеличивается |
| Resize ID | Drag влево | Ширина не меньше минимальной |
| Resize actions | Drag вправо | Иконки остаются кликабельными |
| Hover строки | Любая строка | Нет агрессивной красной заливки |
| Empty state | Нет строк | Текст занимает всю ширину таблицы |

## Dropdown и z-index

| Элемент | Где | Что проверить |
|---|---|---|
| `Все статусы` | Верхняя карточка payments | Список выше таблицы |
| `На странице` | Header таблицы payments | Список выше таблицы |
| Column filter | Header таблицы payments | Меню выше строк |
| `Нотариус` | Верхняя панель orders | Список выше таблицы |
| `На странице` | Header таблицы orders | Список выше таблицы |
| `Тип платежа` | Форма платежа | Не перекрывается соседними полями |
| `Статус` | Форма платежа | Не перекрывается соседними полями |
| `Метод оплаты` | Форма платежа | Не имеет красной системной заливки |
| `Связанная заявка` | Форма платежа | Показывает понятную связь |
| `Связанная подписка` | Форма платежа | Показывает понятную связь |
| `Транзакция` | Форма платежа | Показывает понятное состояние |

## CSV

| Проверка | Ожидание |
|---|---|
| BOM | Excel корректно читает русские заголовки |
| Separator | Используется `;` |
| Header | Колонки идут в стабильном порядке |
| Date | Дата форматируется для `ru-RU` |
| Amount | Сумма с двумя знаками и валютой |
| Fee | Пустая комиссия становится `0,00` |
| Method | Метод человекочитаемый |
| Transaction | Пустая транзакция становится dash |
| Receipt | Имя файла попадает в колонку |
| Relation | `assessmentId` приоритетнее `subscriptionId` |
| Quotes | `"` экранируется двойной кавычкой |
| Semicolon | `;` остается внутри ячейки |
| Empty export | Файл не создается, ошибка видна |
| Failed object URL | Ошибка видна, UI не зависает |
| Log summary | В логе есть rows/status counters |

## Чеки

| HTTP | Сценарий | UI |
|---:|---|---|
| 200 | Storage вернул HTML | Открыть вкладку |
| 200 | Storage вернул файл | Скачать файл |
| 401 | Нет токена | Ошибка авторизации |
| 403 | Нет доступа | Ошибка доступа |
| 404 | Файл отсутствует | Чек не найден |
| 409 | Чек формируется | Повторить позже |
| 503 | Storage недоступен | Временная недоступность |
| fallback | Seed HTML | Backend может вернуть HTML из платежа |

## Audit и notifications

| Событие | Где смотреть | Что важно |
|---|---|---|
| `payment.created` | Audit | Есть userId, paymentId |
| `payment.updated` | Audit | Есть old/new status |
| `payment.receipt.opened` | Audit | Есть fileName |
| `payment.receipt.failed` | Audit | Есть failureReason |
| `robokassa.result.received` | Audit | Есть InvId и OutSum |
| `robokassa.result.succeeded` | Audit | Есть paymentId |
| `robokassa.result.failed` | Audit | Ошибка не ломает webhook |
| Payment created | Notifications | User/Admin получают понятный текст |
| Robokassa success | Notifications | Пользователь видит подтверждение |
| Robokassa failed | Notifications | Админ видит проблему |

| Команда | После каких изменений |
|---|---|
| `nx test admin --skip-nx-cache` | UI payments/orders/monitoring |
| `nx test billing --skip-nx-cache` | Robokassa, receipts, payment services |
| `nx test audit --skip-nx-cache` | Audit events |
| `nx test notification --skip-nx-cache` | Notification fanout |
| `git diff --check` | Перед каждым commit |
| `git status --short --branch` | Перед commit и push |
