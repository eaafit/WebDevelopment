### 2.3. API-методы и endpoints
**Список обязательных API-методов:**

| Метод | Endpoint | Описание | Параметры | Ответ |
|-------|----------|----------|-----------|--------|
| POST | `/api/auth/register` | Регистрация пользователя | `RegisterDto` | `UserWithToken` |
| POST | `/api/auth/login` | Вход в систему | `LoginDto` | `UserWithToken` |
| POST | `/api/auth/logout` | Выход из системы | (token) | `{success: boolean}` |
| POST | `/api/auth/forgot-password` | Запрос восстановления | `ForgotPasswordDto` | `{success: boolean}` |

| GET | `/api/auth/me` | Получить текущего пользователя | (token) | `User` |

**DTO модели:**

- `LoginDto`: email, password
- `ForgotPasswordDto`: email
- `ResetPasswordDto`: token, newPassword

## Раздел 3. UI компонента (≈5 стр.)

### 3.1. Требования к интерфейсу
**Обязательные экраны:**
1. **Страница регистрации** — форма с полями: email, пароль, подтверждение пароля, ФИО, телефон
2. **Страница входа** — форма с email и паролем, ссылка "Забыли пароль?"
3. **Страница восстановления пароля** — ввод email для отправки ссылки
4. **Страница сброса пароля** — ввод нового пароля по токену из email

**Функциональные требования:**
- Валидация email (формат, уникальность)
- Валидация пароля (мин. 8 символов, буквы+цифры)
- Подтверждение пароля (совпадение полей)
- Валидация телефона (российский формат)
- Капча или rate limiting для защиты от ботов

### 3.2. Бизнес-правила
**Для всех ролей (Applicant, Notary, Admin):**
1. Регистрация доступна только с подтверждением email
2. После регистрации требуется активация аккаунта (по email или администратором)
3. Неактивные аккаунты не могут входить в систему
4. Сброс пароля возможен только для активных аккаунтов
5. Токен восстановления действителен 24 часа

**Ограничения доступа:**
- Регистрация доступна: Гость
- Вход доступен: Заявитель, Нотариус, Админ (активные)
- Восстановление пароля: все зарегистрированные пользователи

### 3.3. Скриншоты UI
**Требования к скриншотам:**
1. Скриншот формы регистрации с заполненными данными
2. Скриншот формы входа с ошибкой (неверный пароль)
3. Скриншот страницы восстановления пароля
4. Скриншот успешной регистрации/входа

### 3.4. Руководство пользователя
**Пошаговая инструкция регистрации:**
1. **Шаг 1:** Переход на страницу регистрации
   - Пользователь нажимает "Зарегистрироваться" на главной странице
   - Система отображает форму регистрации
2. **Шаг 2:** Заполнение данных
   - Пользователь вводит email, пароль, ФИО, телефон
   - Система проверяет валидность данных в реальном времени
3. **Шаг 3:** Подтверждение email
   - Пользователь получает письмо со ссылкой активации
   - После перехода по ссылке аккаунт активируется

**Типичные сценарии:**
- Сценарий 1: Успешная регистрация и вход
- Сценарий 2: Восстановление забытого пароля
- Сценарий 3: Попытка входа с неверными данными
