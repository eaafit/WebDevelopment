## Skill: Поднять dev‑окружение и сделать smoke‑проверку

### Когда использовать

- Нужно воспроизвести окружение локально перед разработкой/отладкой.

### Шаги

1. **Установи зависимости**:
   - `pnpm install`
2. **Проверь Docker**:
   - `docker info`
3. **Подними инфраструктуру**:
   - `docker-compose up`
4. **Запусти frontend**:
   - `pnpm nx serve web`
5. **Если запускаешь API на хосте** (не в docker):
   - убедись, что `S3_ENDPOINT` указывает на `http://127.0.0.1:<MINIO_PORT>`
   - переменные MinIO/S3 должны соответствовать `.env.example`
6. **Smoke‑проверка observability**:
   - у API доступны `/health` и `/metrics` (если API запущен)
   - при поднятой Grafana доступны готовые дашборды (если включены сервисы в compose)

### Готово когда

- Инфраструктура поднята без ошибок.
- Web приложение доступно и работает базовая навигация.
- (Если запускали API) `/health` и `/metrics` отвечают.

### Источники в репозитории

- `docs/developer-setup.md`
- `docker-compose.yaml`
- `.env.example`
