## Skill: Включить структурированное ops‑логирование без утечек секретов/PII

### Когда использовать

- Нужны технические логи для диагностики (не продуктовый аудит): JSON, request id, уровни логирования.

### Backend (NestJS, `apps/api`) — шаги

1. **Выбери библиотеку**: `pino` + `nestjs-pino` (или эквивалент) под NestJS 11.
2. **Единый JSON‑лог в stdout** в production‑like режиме.
3. **Логируй входящие запросы** (HTTP/Connect): метод, путь, статус, длительность.
4. **Correlation/request id**:
   - принимать из заголовка (`X-Request-Id` или `traceparent`) или генерировать,
   - добавлять в каждую строку логов запроса.
5. **Политика безопасности логов**:
   - по умолчанию не логировать bodies,
   - маскировать/allowlist чувствительные поля.
6. **Уровни логирования из env** (`LOG_LEVEL`/`NODE_ENV`) и задокументировать переменные.

### Frontend (Angular) — шаги

1. **Единый сервис логирования** (обёртка): dev — удобно, prod — минимум шума.
2. **Глобальный `ErrorHandler`**:
   - перехватывает необработанные ошибки,
   - пишет в единый формат (например `service`, `env`, опционально `userId` без PII).
3. **Запрет утечек**:
   - не писать в логи значения из `localStorage`/токены/PII.

### Готово когда

- В API в stdout идут структурированные JSON‑логи с request id.
- Запросы логируются без тел/секретов.
- На фронте необработанные ошибки проходят через единый обработчик.
