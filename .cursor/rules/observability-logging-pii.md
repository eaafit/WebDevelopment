## Observability: ops‑логи vs продуктовый аудит + политика PII

### Правило

- **Не смешивай “ops‑логи” и “продуктовый аудит”**:
  - ops‑логи: диагностика процесса, ошибки, latency, request id (infra/observability)
  - аудит: доменные события для UI/таблицы аудита (admin monitoring)
- **По умолчанию не логируй** тела запросов/ответов и поля, содержащие:
  - токены/пароли/секреты,
  - платёжные данные,
  - персональные данные (PII).
- **Correlation / request id обязателен**: принимать из заголовка или генерировать и пробрасывать в логи запроса.

### Почему

- Логи должны помогать диагностике и не становиться утечкой данных/секретов; аудит — отдельный продуктовый слой.
